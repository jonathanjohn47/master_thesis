Empirical Analysis of Accuracy–Privacy Trade-offs in Federated Learning for Mobile Movie Recommendation Systems

Introduction

In today’s world, where digital platforms have become ubiquitous, recommendation systems are becoming a necessity more and more to cater to the needs of individual users. This recommendation requires a lot of user’s data to understand user’s preferences and choices so that content relevant to the user are displayed to the user. This is taking towards to a growing concern for user’s data privacy because in order to train the recommendation Machine Learning model, user data has to be fed in. Once we fetch user’s personal data, we breach the privacy; and the situation needs a solution or a workaround. Many regulatory frameworks, like the General Data Protection Regulation and the California Consumer Privacy Act, that deal with the privacy of users in these recommendation systems propose some methods to preserve user’s privacy while still catering to the user’s individual recommendation needs. One of these methods is Federated Learning (Ammad-ud-din et al., 2019). 

Federated Learning is a method in which the machine learning models get trained in a decentralized way. Which means, models are trained on individual devices of each individual users instead of bringing all user’s sensitive data to the central server where the whole point of privacy is killed (Bonawitz et al., 2019). This technique of Federated Learning where models are trained on individual devices ensures that no sensitive data goes out of the user’s device for any purpose; let alone centralized machine learning model training. This approach achieves two aspects:





User’s sensitive data is secured from coming out of the user’s device.



User is still catered with personalized recommendation content (Ramaswamy et al., 2020).

The greatest achievement of this technique of Federated Learning is that the user’s sensitive data remains in the user’s device and never leaves the device whatsoever. This builds trust among users that their data is secure while they still enjoy personalized recommendation content (Jiang et al., 2024). The data which we are concerned about, and that must be secured, comes in the form of ‘likes’, ‘favorites’, ‘views’, ‘clicks’, or any other type of interaction with the content that directly or indirectly reflects the user’s preferences and choices. There have been many techniques and approaches that have been proposed to keep user’s data private while still giving them personalized recommendations of content. Some of these techniques are 





Differential Privacy: Differential Privacy is a mathematical framework which makes sure that the output of the trained model is almost same with the user’s data included in the dataset or without it (Abadi et al., 2016).
It is quantified by $\epsilon$ (privacy loss. Lower value of $\epsilon$means stronger is the privacy) and small $\delta$ (failure probability, which means that a small probability that the differential privacy guarantee may not apply (Fioretto et al., 2024, p. 11; Ranaweera et al., 2025, p. 3)).



Homomorphic Encryption: This technique first encrypts the user’s data into what is called “ciphertexts”. Then it executes computations to be performed on the ciphertexts; instead of the raw data. The results of these computations is another ciphertext which when decrypted is very similar to the results if the computations were performed on raw, unencrypted data (Kholod et al., 2024, p. 3).



Secure Multi-Party Computation: This method works in a way that the central server only sees the aggregated result and never the individual data. This is achieved through adding noise to individual data point in such a way that when aggregation happens in the central server, the noises cancel out and an aggregated result is obtained while at the same time, the central server misses out on what the individual data was (Bonawitz et al., 2016).




Despite all these algorithms and protocols, Federated Recommender system is still not 100% perfect. The system is still prone to attacks like inference attacks. Inherent heterogeneity of user data can also degrade the recommendation output and performance (Hu & Song, 2024). It is important that a balance between maintaining high recommendation accuracy and making sure the data remains private as much as possible, is maintained. Especially when it comes to maintaining privacy in data which is Non-Independent and Identically Distributed (non-IID) along with heavy load of model constantly updating from time to time (Abadi et al., 2016). This paper aims to analyse the trade-off between accuracy and privacy in federated learning specifically focused on data which recommends movies on mobile phones. In this paper, we will also check how good federated learning movie recommenders work under various privacy setups. Especially, we will analyze how tweaks in differential privacy affect accuracy and user experience. This paper will also analyze the extra computing and network costs of these privacy tools to see how they perform on real mobile phones.

This tension requires evaluating privacy mechanisms in federated recommenders not only for effectiveness but also fairness across user groups and data distributions (Balbierer et al., 2024, p. 12).



This calls for adaptive mechanisms. They dynamically tweak privacy budgets to cut disparities while upholding privacy (Zhang et al., 2023, p. 994). Plus, they allocate privacy resources smartly—without weakening differential privacy (L. Wang et al., 2024, p. 12; Q. Wang et al., 2021, p. 890). Federated methods like FedSlate deliver uneven gains, unfairly favoring those short on user feedback (Deng et al., 2024, p. 13). Fairness gaps, data variety, privacy-utility balances, and smart adaptations demand better solutions. One promising direction involves user-consented federated recommendation systems that allow individuals to define their privacy preferences, thereby mitigating unnecessary accuracy loss from over-protection while also preventing information leakage (Hu & Song, 2024). Further exploration into dynamic privacy mechanisms is essential to optimize this balance, ensuring that privacy-preserving techniques do not inadvertently compromise the utility and fairness of recommendation systems across diverse user populations and data environments (Akkaya, 2025, p. 20). Furthermore, addressing data scarcity in federated learning environments for recommendation systems, especially where user interaction data is limited, necessitates the deployment of data augmentation techniques and knowledge transfer from richer datasets to enhance model robustness and effectiveness (Li & Long, 2024, p. 6).

This section reviews existing literature on federated learning for recommendation systems, with a particular focus on privacy-preserving mechanisms and their impact on model performance (Sun et al., 2022). It also examines various approaches to addressing challenges such as data heterogeneity, communication efficiency, and fairness in federated recommender systems, while considering how models handle rating bias effectively (Gao et al., 2025; Kaur et al., 2024). Recent advances in federated learning for recommendation systems, such as FedRec and FCF, have primarily focused on collaborative filtering based on matrix factorization, with FedMF further incorporating homomorphic encryption for secure gradient transmission (Nguyen et al., 2024). Despite these advancements, many existing studies often neglect the multifaceted aspects of fairness, bias, and threats to system integrity, which are crucial for developing robust and trustworthy recommender systems (Roy et al., 2024). While some works aim to enhance the efficiency of federated recommender systems through faster convergence or lightweight communication strategies (Yuan et al., 2023), a comprehensive understanding of fairness and integrity under various attack models remains an active research area. Moreover, the integration of foundation models within federated recommendation systems presents a promising avenue for enhanced performance and privacy, despite being in its nascent stages (Li & Long, 2024). However, to fully realize the potential of foundation models in federated recommendation systems, it is essential to explore how their pre-training capabilities can mitigate data heterogeneity challenges and improve model aggregation efficiency (Li & Long, 2024). The application of large language models within this federated context, as detailed in recent literature, suggests a paradigm shift from simple data integration to fully leveraging LLMs for improved performance, albeit with increased communication costs due to large parameter sizes (Zhao et al., 2024).

This integration also introduces novel challenges concerning privacy-security trade-offs and handling non-IID data distributions inherent in federated settings (Li & Long, 2024). Furthermore, while federated learning offers privacy benefits, current federated recommendation systems still face limitations, such as difficulties in handling non-IID data distributions and model performance degradation due to privacy-preserving mechanisms (Luo et al., 2023). Addressing these challenges requires a shift towards more sophisticated federated learning frameworks that can effectively adapt to diverse data characteristics and maintain high recommendation accuracy while upholding stringent privacy standards (Li & Long, 2024). The development of frameworks that ensure fair and unbiased recommendations, protect user privacy, and provide transparency in the generation and dissemination of results is crucial for future recommender systems (Chen et al., 2024). Privacy-Preserving Mechanisms in Recommender Systems

In federated learning for recommender systems, achieving personalization while upholding privacy necessitates sophisticated mechanisms beyond simple data encryption (Jiang et al., 2024). This includes exploring advanced differential privacy techniques and secure multiparty computation to protect user data during model training and inference (Fan et al., 2025). Specifically, homomorphic encryption and zero-knowledge proofs offer robust frameworks for ensuring privacy without sacrificing the utility of aggregated model updates (Luo et al., 2024). Furthermore, integrating these techniques with personalized federated learning approaches can enable the creation of robust and interpretable personality representations for individual users while inherently preserving privacy (Li et al., 2025). This is especially important given the systemic issues of data sparsity and statistical heterogeneity across individual clients, which can impede effective inference of nuanced user traits (Li et al., 2025). Moreover, the dynamic nature of user preferences further complicates the task of building accurate recommendation models, necessitating continuous adaptation and learning from evolving data distributions (Akkaya, 2025). A core challenge remains in defining what constitutes "personality leakage" and establishing appropriate utility metrics that account for deep personalization, given that traditional privacy frameworks often fall short in capturing semantic privacy requirements for complex inferred personality attributes (Li et al., 2025). This necessitates the development of multi-task, multi-domain solutions that are robust and versatile enough to handle the complexity and variability inherent in real-world applications (Luo et al., 2024). The sophisticated representational capacity of federated foundation models, inherited from traditional foundation models, shows particular promise for modeling intricate client attributes like personality within recommendation systems, even under the data constraints common in distributed environments (Li et al., 2025). However, despite their potential, existing metrics for quantifying and auditing personality privacy within these federated foundation models remain inadequate, posing a significant hurdle for robust evaluation (Li et al., 2025). Therefore, developing advanced objective functions and optimization strategies that explicitly embed personality preservation goals into the federated foundation model refinement process is vital (Li et al., 2025). Further research is needed to devise methodologies that can accurately measure the trade-offs between personalization accuracy and the degree of personality leakage, particularly within complex, high-dimensional user representations (Cooper & Geller, 2025). A primary concern revolves around the potential for these sophisticated models to infer sensitive user attributes, such as personality traits, from seemingly innocuous interactions, raising ethical considerations regarding data misuse and algorithmic bias (Celli et al., 2025). Addressing these ethical considerations requires the implementation of transparent auditing mechanisms and the establishment of clear governance frameworks to regulate the use and dissemination of inferred user characteristics. Moreover, this necessitates a critical examination of how federated foundation models handle test-time distribution shifts, as conventional methods may fall short in adapting to complex scenarios due to their need to train all parameters, thereby failing to effectively manage shifts in user data dynamics during ongoing operation (Yang et al., 2024). 

This underscores the importance of developing adaptive, privacy-preserving personalization strategies within federated learning that can dynamically adjust to evolving user preferences while maintaining robust privacy guarantees (Borazjani et al., 2025; Li et al., 2025). Accuracy-Privacy Trade-offs in Machine Learning

The inherent tension between achieving high model accuracy and preserving user privacy is a foundational challenge in developing effective federated learning systems for mobile movie recommendation, necessitating careful consideration of various trade-off strategies. Specifically, while federated learning inherently offers privacy benefits by keeping raw data on local devices, it remains susceptible to advanced threats such as model inversion attacks and data poisoning, necessitating the integration of additional privacy-preserving mechanisms (Okolo et al., 2025). These mechanisms often introduce a compromise in model performance or increased computational overhead, thereby exacerbating the accuracy-privacy trade-off (Chen et al., 2024). For instance, integrating differential privacy, while providing quantifiable privacy guarantees, often requires adding noise to gradients or model updates, which can degrade model accuracy, especially with limited data or complex models (Li et al., 2025). Similarly, secure multiparty computation and homomorphic encryption offer strong privacy assurances but often incur substantial computational costs, rendering them impractical for real-time mobile recommendation systems (Hallaji et al., 2024).

The selection of appropriate privacy-enhancing technologies must therefore be carefully balanced against the acceptable performance degradation and resource constraints of the mobile environment (Monteiro et al., 2024; Zhang et al., 2024). The fundamental expression governing this trade-off highlights that simultaneously minimizing privacy leakage and utility loss is impossible when interacting with large language models, indicating that a balance must be struck (Fan et al., 2025). This balance is particularly challenging in mobile recommendation systems, where limited computational resources and the need for real-time inference constrain the complexity of privacy mechanisms that can be deployed (Ahmadi et al., 2025). Consequently, understanding the context-dependent nature of these trade-offs is crucial for designing federated learning systems that uphold principles of responsible AI, ensuring fairness, privacy, and equitable real-world applications (Wasif et al., 2025). Furthermore, the deployment of differential privacy, though beneficial for enhanced user privacy, can disproportionately impact underrepresented user groups, thereby necessitating techniques to mitigate this fairness concern (Wasif et al., 2025). Moreover, the adoption of spectral robust averaging, while theoretically optimal for ensuring both robustness and privacy, presents practical implementation difficulties, suggesting a need for more efficient alternatives (Zangana et al., 2024). 

This constraint often forces a careful evaluation of the privacy budget, ensuring that the level of noise introduced by differential privacy, for example, is sufficient to protect sensitive user data without rendering the model's recommendations irrelevant (Li & Long, 2024; Shirvani et al., 2024). Furthermore, the dynamic and volatile nature of user preferences and device participation in mobile federated learning environments compounds this challenge, as privacy mechanisms must adapt to continually changing threat landscapes and data distributions (Prigent et al., 2024). Achieving strong privacy guarantees often necessitates a trade-off with model utility, as increased noise to protect sensitive data can degrade performance, especially in complex machine learning models (Aguilera-Martínez & Berzal, 2025). This phenomenon, often referred to as "no free lunch" in privacy-preserving machine learning, underscores the inherent difficulty in simultaneously optimizing all system qualities such as privacy assurance, model performance, and efficiency (Xu et al., 2024). For instance, incorporating differential privacy mechanisms often leads to a reduction in model accuracy, as the perturbation introduced to safeguard privacy can distort the underlying data patterns and relationships (Xu et al., 2022). This necessitates a meticulous calibration of the privacy budget, balancing the need for robust privacy protection with the maintenance of acceptable model utility for effective movie recommendations. This delicate balance is further complicated by the trade-off between privacy and fairness, where stringent privacy measures can inadvertently exacerbate demographic disparities by limiting information learned about under-represented groups, potentially leading to biased recommendations (Arasteh et al., 2024). Key manifestations of this privacy-fairness tension include:





Disproportionate impacts from differential privacy mechanisms like gradient clipping and noise addition, which degrade accuracy more severely for underrepresented classes or subgroups (Ren et al., 2024);



Amplified utility loss for marginalized subpopulations, as noise exacerbates biases and leads to the "poor get poorer" effect (Cummings et al., 2024; Saha et al., 2024).

This necessitates the development of adaptive privacy-preserving techniques that can dynamically adjust the level of privacy protection based on the sensitivity of the data and the specific characteristics of different user groups, thereby mitigating unintended fairness compromises (Khezresmaeilzadeh et al., 2025). The challenge is further compounded in real-world applications by the heterogeneity of data and non-IID distributions across various mobile devices, requiring more sophisticated approaches to balance privacy and utility (Farooqi et al., 2025). One proposed solution involves adaptive mechanisms that adjust privacy parameters based on the utility impact, aiming to optimize the trade-off in dynamic environments (Meziani et al., 2020). This ensures that while strong privacy guarantees are maintained for sensitive data, model utility is maximized for all user groups, including those that are typically underrepresented (Arasteh et al., 2024). The "no free lunch" theorem in federated learning further elucidates this challenge, asserting that enhancing privacy inevitably incurs some cost in terms of model utility (Zhang et al., 2022). This often translates into a privacy-utility trade-off where increased privacy leads to information loss and a higher re-identification probability, impacting prediction accuracy (Roy et al., 2024). Therefore, advanced techniques, such as Rényi Differential Privacy and adaptive privacy budget allocation, are crucial for mitigating this utility loss while upholding privacy in federated learning systems (Farooqi et al., 2025). This highlights the need for sophisticated frameworks that can dynamically manage privacy budgets in response to varying data sensitivities and utility requirements across diverse client populations (Nguyên et al., 2024; Wang et al., 2021). Moreover, the introduction of differential privacy can disproportionately affect the utility for marginalized or underrepresented subpopulations, as the noise addition may exacerbate existing biases and lead to a greater utility loss for these groups (Cummings et al., 2024; Saha et al., 2024).

This phenomenon, where "the poor get poorer," underscores the critical need for fairness-aware privacy-preserving mechanisms in federated learning that specifically address disparate impacts on sensitive subgroups (Cummings et al., 2023; Kairouz et al., 2021). For instance, Personalized DP can customize privacy budgets based on individual client data sensitivity and contributions, aiming to maximize utility while ensuring strong privacy for sensitive datasets (Farooqi et al., 2025). Furthermore, approaches like uncertainty-aware frameworks attempt to balance privacy and fairness by understanding how different noise mechanisms affect various client groups (Wasif et al., 2025). This complexity is further intensified by the inherent tension between privacy and fairness, as strengthening privacy protections can sometimes inadvertently lead to biased outcomes for certain demographic groups (Sun et al., 2025). This can occur because fairness constraints typically necessitate global knowledge of demographic distributions, which conflicts with FL's distributed nature and limited information sharing capabilities (Sun et al., 2025). Consequently, novel methods are emerging to address this challenge by integrating fairness metrics into the privacy-preserving mechanisms themselves, such as through multi-objective optimization that simultaneously considers accuracy, privacy, and fairness constraints (Chen et al., 2023; Sun et al., 2025). Moreover, selecting appropriate differential privacy methods and training parameters is critical, as certain choices, such as those relying on Poisson subsampling, can lead to variable-sized mini-batches that strain memory on low-end devices, potentially excluding them from participation (Ahmadi et al., 2025). This exclusion can exacerbate existing disparities and lead to a less representative global model, highlighting the necessity of fairness-aware client selection strategies (Kairouz et al., 2021; Wang et al., 2024). Addressing these challenges requires a comprehensive framework that considers both privacy and fairness across all stages of federated learning, from client selection to model aggregation, to mitigate algorithmic biases and ensure equitable outcomes for all participants (Ferry, 2023; Wasif et al., 2025). This is particularly crucial given that certain privacy mechanisms, while protecting individual data, can inadvertently contribute to the "Matthew effect," where models perform worse for clients with limited resources, thereby widening performance gaps (Gao et al., 2023). This intricate interplay necessitates the development of sophisticated methodologies that explicitly account for both privacy and fairness, moving beyond solutions that address these ethical considerations in isolation (Rafi et al., 2023). Indeed, several studies acknowledge the intricate relationship between privacy and fairness, recognizing that an isolated approach to either can undermine the overall integrity and ethical standing of federated learning systems (Ling et al., 2024; Rafi et al., 2023). For instance, an overemphasis on privacy can lead to the neglect of fairness concerns, as stringent privacy measures may obscure the very biases they are meant to address within the aggregated model (Wang et al., 2024; Wasif et al., 2025). 

Methodology

In this section, we delineate the systematic methodology employed throughout this research, encompassing the rigorous approaches, advanced techniques, specialized tools, and meticulously designed procedures utilized to comprehensively investigate the core research questions. Central to this thesis, the methodology articulates a holistic experimental framework tailored to dissect the complex accuracy–privacy–fairness trade-offs prevalent in federated learning deployments for mobile movie recommendation systems(Ammad-ud-din et al., 2022; Maeng et al., 2022). This framework systematically addresses a suite of critical inquiries, including: the selection and preprocessing of datasets to simulate realistic heterogeneous mobile environments; the implementation of suitable models and algorithms optimized for resource-constrained devices and recommender tasks; the configuration of FL protocols accounting for non-IID data distributions and communication constraints; the integration and empirical evaluation of privacy-preserving mechanisms such as differential privacy and secure aggregation; the precise measurement of computational, memory, and communication resources to assess feasibility on edge devices; and the deployment of comprehensive evaluation metrics to quantify trade-offs among privacy guarantees (e.g., $\epsilon$-DP budgets), fairness disparities across user subgroups, and recommendation accuracy (e.g., NDCG, Recall@K), particularly under the sparsity and personalization demands of mobile movie streaming contexts(Nivethitha & Aghila, 2024; L. Wang et al., 2023; Q. Wang et al., 2021).

To facilitate reproducibility, scalability, and alignment with state-of-the-art FL benchmarking practices(Lai et al., 2021; Morafah et al., 2023), the methodology is organized into six interconnected components that collectively enable robust analysis of these trade-offs amid data and system heterogeneity(Abdelmoniem et al., 2022; Baumgart et al., 2024): Data Preparation and Partitioning, leveraging MovieLens-1M for non-IID horizontal partitioning(Ammad-ud-din et al., 2022; Nivethitha & Aghila, 2024); Model Architectures, featuring federated collaborative filtering and matrix factorization adapted for mobile efficiency(Lin et al., 2020; Wu et al., 2023); Federated Learning Setup and Training, employing FedAvg with adaptive client selection(Wang et al., 2023); Privacy Protection Implementation, incorporating DP noise mechanisms and budget allocation(Farooqi et al., 2025); Resource Measurement, profiling CPU/GPU usage, memory, and bandwidth on heterogeneous devices(Ahmadi et al., 2025); and Evaluation Metrics and Trade-off Analysis, integrating utility, fairness (e.g., demographic parity), and privacy metrics(Rafi et al., 2023; Wasif et al., 2025).

The methodology is organized into six main components that together form a comprehensive experimental framework:





Data Preparation and Partitioning: This component details the dataset selection, preprocessing steps, and data partitioning strategies employed to simulate realistic federated learning scenarios for mobile movie recommendation systems. The MovieLens-1M dataset(Ammad-ud-din et al., 2022; Nivethitha & Aghila, 2024) is selected as the primary benchmark, comprising 1,000,209 explicit ratings (scale 1-5) from 6,040 users on 3,952 movies collected in 2000, augmented with user demographics (age, gender, occupation, zip code) to enable realistic personalization and heterogeneity simulation(Ammad-ud-din et al., 2022; Neumann et al., 2023; Nivethitha & Aghila, 2024). This dataset is widely adopted in FL recommender systems due to its sparsity (~95%), diverse user behaviors, and relevance to mobile streaming contexts where ratings represent watch history or content requests(Ammad-ud-din et al., 2022; Nivethitha & Aghila, 2024). Preprocessing begins with filtering: users and movies with fewer than 5 interactions are removed to address sparsity and cold-start issues, reducing the dataset while preserving core interactions(Malik et al., 2020). Explicit ratings are then transformed into implicit feedback by binarizing (ratings ≥ 4 as 1/positive interaction, others as 0), simulating binary "watch/like" signals common in mobile apps, with missing entries as unobserved(Ammad-ud-din et al., 2022). Timestamps are used for chronological train-test splits (80/20), ensuring temporal realism for sequential recommendations(Han et al., 2024). Additional normalization standardizes ratings per user to handle scale variations. To mimic FL on heterogeneous mobile clients, data is partitioned horizontally (user-centric): users are randomly assigned to K=100 clients, each holding all interactions of its assigned users/items, naturally inducing non-IID distributions from varying user preferences, data volumes, and activity levels (e.g., power-law distribution of interactions per user)(Maeng et al., 2022; Wang et al., 2023; Wei et al., 2024). Client dataset sizes vary (e.g., 10-200 users/client) to simulate mobile data heterogeneity, with ~45-80% sparsity retained(Malik et al., 2020; Wang et al., 2023). This setup ensures privacy (no raw data sharing), scalability for edge devices, and evaluation of accuracy-privacy trade-offs under realistic non-IID conditions(Ammad-ud-din et al., 2022; Nivethitha & Aghila, 2024).



Model Architectures: Details the machine learning models used (matrix factorization), their mathematical formulation, and implementation choices that make them suitable for mobile deployment. This includes adaptations for resource-constrained environments, such as quantization and model pruning, alongside considerations for efficient inference on diverse mobile hardware. Furthermore, this section will elaborate on how these models are specifically tailored for recommender systems, including collaborative filtering and matrix factorization approaches, to ensure high relevance and performance within the federated learning paradigm (Nivethitha & Aghila, 2024). This also encompasses the integration of novel architectures designed for federated settings, such as those employing federated collaborative filtering or federated matrix factorization, which are optimized for decentralized training and privacy preservation (Ammad-ud-din et al., 2022; Wei et al., 2024; Wu et al., 2023). The mathematical formulations for these models will be presented, emphasizing how their architectures facilitate privacy-preserving mechanisms, such as secure aggregation or differential privacy, without significantly compromising model utility (“A Privacy-Preserving Federated Recommender System with Neuro-Fuzzy Modeling and Local Differential Privacy,” 2025; Lin et al., 2020). The selection and adaptation of these models are critical for ensuring that the proposed federated learning framework is both computationally feasible on mobile devices and capable of capturing complex user-item interactions for accurate movie recommendations (Lin et al., 2020). These architectural choices are meticulously evaluated for their robustness against non-IID data distributions and their capacity to maintain performance under varying communication constraints inherent in mobile FL environments (Maeng et al., 2022). Additionally, the methodologies incorporate mechanisms for client-side adaptation, allowing individual devices to fine-tune global models with their local data, thereby enhancing personalization while respecting privacy boundaries (Jiang et al., 2024; Li & Long, 2024). The discussion will also cover the rationale behind adopting a multi-global model strategy within a hierarchical framework to address heterogeneity in devices and data distributions, thereby enhancing scalability and personalization across the network (Bashir et al., 2024). This approach enables personalized and efficient learning for diverse devices and data distributions, diverging from traditional FL paradigms by implementing a multi-global model strategy within a hierarchical framework (Bashir et al., 2024). This multi-global model strategy contrasts with conventional federated learning approaches that typically maintain a single global model, offering a more nuanced approach to handling diverse client characteristics and preferences (Jiang et al., 2024). This allows for a more adaptive and resilient system, particularly beneficial in environments with significant user heterogeneity and varied device capabilities. 



Federated Learning Setup and Training: Explains the federated learning protocol (FedAvg), communication mechanisms, training procedures, and how clients participate in the distributed training process. This section will detail the FedAvg algorithm as the foundational approach, alongside specialized algorithms designed to manage non-IID data through multi-global models within a three-tier federated learning architecture (Bashir et al., 2024). This hierarchical framework enhances data privacy, computational efficiency, and scalability compared to traditional client-server FL models (Bashir et al., 2024). Specifically, the three-layered architecture integrates clients, an edge layer, and a federated edge layer, facilitating efficient aggregation and personalized model management (Bashir et al., 2024). This architecture allows for robust handling of non-IID data characteristics by enabling aggregation across multiple layers and supporting multiple global models, which significantly improves model efficacy and addresses data and computational heterogeneity (Bashir et al., 2024). The edge layer, acting as a crucial coordinator, analyzes metadata from client training sessions to assess model states and requirements, querying the federated edge layer if a suitable model is not locally found (Bashir et al., 2024). This multi-tiered approach allows for more granular control over model aggregation and distribution, adapting to the dynamic and diverse nature of mobile movie recommendation systems (Bashir et al., 2024; Hamood et al., 2024; Wang et al., 2022). The fedge layer, in turn, is tasked with storing and managing these distinct global models, ensuring their currency and representativeness of various data landscapes (Bashir et al., 2024). This structured orchestration facilitates optimal model selection and distribution, which is particularly advantageous for managing the varying data distributions and computational capacities of mobile clients in a movie recommendation system (Bashir et al., 2024). This comprehensive approach ensures that both global insights and personalized preferences are effectively captured while maintaining user privacy and system efficiency (Bashir et al., 2024; Wu et al., 2023). The communication between clients and the server leverages established algorithms like FedAvg, with the flexibility to incorporate custom aggregation strategies to meet specific research needs (Jung et al., 2024). The server aggregates these local updates to refine the global model, disseminating the improved version back to clients for subsequent training rounds until convergence is achieved (Nivethitha & Aghila, 2024; Tertulino, 2025). This iterative process is fundamental to federated learning, enabling the collaborative training of robust models without centralizing raw user data, thus enhancing privacy (Neumann et al., 2023). This distributed approach ensures that sensitive information remains on individual devices, aligning with stringent data protection regulations and user expectations regarding privacy (Saif et al., 2025; Zhao et al., 2025). 



Differential Privacy Implementation: Describes how privacy protection is added through DP-SGD, including gradient clipping, noise addition, and privacy budget accounting using Renyi Differential Privacy. Furthermore, the integration of techniques like gradient sparsification and differential coding will be explored to optimize communication overhead during local model updates, ensuring efficient transmission of only significant parameter changes (Li et al., 2025). This layered defense mechanism significantly bolsters the privacy guarantees beyond basic DP, making the system more resilient to various inference attacks while optimizing resource utilization (Farooqi et al., 2025; Li et al., 2024). These strategies are crucial for maintaining the delicate balance between rigorous privacy protection and acceptable model performance, particularly in resource-constrained mobile environments (Luo et al., 2023). Furthermore, the proposed approach integrates local differential privacy mechanisms directly into the training pipeline to further enhance privacy protection (Na et al., 2025). This is accomplished by adding carefully calibrated noise to the client updates before they are transmitted to the server, ensuring that individual contributions are indistinguishable while still contributing to model learning (Talaei & Izadi, 2024). This mechanism, primarily through DP-SGD which incorporates per-sample gradient clipping to bound the sensitivity of individual updates and subsequent addition of calibrated Gaussian noise drawn from a distribution scaled to the privacy budget (tracked via Renyi Differential Privacy for tighter composition bounds), ensures that the aggregated global model captures overarching patterns across the federation without memorizing or overfitting to any specific user's private data, even during the central server's aggregation phase (Farooqi et al., 2025; Wu et al., 2023). Gradient clipping limits the L2-norm of each sample's gradient, preventing any single client's update from disproportionately influencing the global model, while the noise injection renders individual contributions statistically indistinguishable from one another or from random perturbations, thereby providing formal $(\epsilon, \delta)$-differential privacy guarantees that bound the privacy leakage regardless of the adversary's access to model outputs or intermediates (Bu et al., 2022; Tayyeh & AL-Jumaili, 2024). This not only thwarts advanced inference threats like membership inference or reconstruction attacks—where adversaries might otherwise extract user-specific movie preferences from gradient patterns—but also empirically enhances model generalization by acting as a form of regularization, as demonstrated in large-scale federated settings where DP-FedAvg yields production-ready models resilient to unintended memorization of rare or sensitive user-item interactions (Ramaswamy et al., 2020; Thakkar et al., 2021).



Privacy Attack Models: Details the adversarial evaluation methods used to assess the effectiveness of privacy protection mechanisms. This includes simulating prominent attacks such as membership inference attacks, attribute inference attacks, data reconstruction attacks, and model poisoning attacks tailored to federated recommender systems, particularly those handling movie recommendations with non-IID data distributions (Mai & Pang, 2023; Rong et al., 2022; Yuan et al., 2023; Zhang et al., 2023). Membership inference attacks, for instance, enable an honest-but-curious server to infer specific user-item interactions by exploiting differences in model behavior or gradients from shadow models trained on synthetic data mimicking client updates (He et al., 2025; Yuan et al., 2023; Zhang et al., 2023). Attribute inference attacks leverage uploaded model parameters to deduce sensitive user demographics or preferences, highlighting vulnerabilities even in privacy-enhanced federated setups (Zhang et al., 2023). Reconstruction attacks by the server reconstruct private ratings or interactions from gradients, achieving over 80% accuracy in matrix factorization scenarios on datasets like MovieLens, underscoring risks in hierarchical FL architectures (Mai & Pang, 2023; Pham et al., 2023). Poisoning attacks, also known as model poisoning or gradient poisoning attacks, involve deploying a small number of malicious clients that simulate synthetic or fake users to upload carefully crafted gradients during federated aggregation rounds (Mozaffari et al., 2024). These adversarial updates manipulate the global model parameters, particularly item embeddings, to artificially boost the exposure and ranking of target movies in top-$K$ recommendations, even without any prior knowledge of benign clients' data distributions or interactions (Rong et al., 2022; Yin et al., 2024; Yuan et al., 2023; Zhang et al., 2021). For instance, attackers approximate benign user embeddings through random sampling or hard mining strategies, then generate poisoned gradients that shift the target item's representation closer to popular movies in the embedding space, exploiting inherent popularity biases in recommender systems (Rong et al., 2022; Zhang et al., 2021). Such attacks are particularly effective in non-IID settings typical of movie recommendation datasets like MovieLens, where a few malicious clients (e.g., 1-5% of total) can significantly elevate unpopular target movies' hit rates and exposure by over 20-50% across users, while maintaining stealth by preserving overall model accuracy (Yin et al., 2024; Yuan et al., 2023). These evaluations quantify the extent of privacy and integrity leakage through standardized metrics, including attack success rates (e.g., target item promotion rate in top-$K$ lists), F1-scores for distinguishing malicious influences, reconstruction fidelity for assessing recoverable private interactions, and model utility degradation (e.g., NDCG drops) (Yu et al., 2023; Yuan et al., 2023). Empirical results validate the resilience of proposed defenses such as DP-SGD—which injects calibrated Gaussian noise to obscure crafted perturbations—gradient sparsification that transmits only significant parameter changes to dilute malicious impacts, and multi-global model strategies that diversify aggregation across heterogeneous client clusters, collectively reducing attack efficacy by 30-70% without substantial accuracy loss (Yuan, Nguyen, et al., 2023; Yuan, Yang, et al., 2023). Such comprehensive evaluations are vital for understanding the practical security boundaries of federated recommender systems, especially when deployed in real-world mobile environments with diverse and potentially adversarial user bases. 



Resource Profiling and Metrics: Details the measurement of key computational resources essential for viable deployment of federated recommender systems on resource-constrained mobile devices, including CPU utilization, memory footprint, energy consumption, communication overhead, and inference latency. These metrics are profiled using specialized tools such as I-Prof for predicting computation time and energy impact of local training tasks on Android devices (Damaskinos et al., 2020), Trepn and Snapdragon Profiler for system-wide power monitoring (Wang et al., 2022), and benchmarks like FedScale which incorporate real-world device traces for computation speed (FLOPs/s), communication throughput (MB/s), and client availability patterns (Lai et al., 2021). In federated movie recommendation scenarios with large item catalogs like MovieLens (e.g., thousands of movies), memory usage is a primary bottleneck: a 128-dimensional item embedding consumes approximately 10.2 KB, rendering full storage of millions of items (>1 GB) infeasible on typical smartphones with 4-8 GB total RAM, necessitating techniques like elastic embeddings or server-side item storage to cap per-client memory at 25-512 MB (Imran et al., 2022a, 2022b; Zheng et al., 2024). Computation is quantified via FLOPs or wall-clock time per local epoch, with acceptable thresholds under 1-5 seconds per round on mid-range CPUs (e.g., 4× vCPU) to avoid user-perceived delays (Chen et al., 2021). Energy profiling targets <0.1% daily battery drain for continuous FL participation, as achieved by systems like FLeet (0.036% per day), balancing privacy mechanisms like DP-SGD—which add Gaussian noise computation overhead—with regularization benefits (Damaskinos et al., 2020; Ramaswamy et al., 2020). Communication metrics track uploaded gradient sizes (e.g., 1-10 MB/round for embedding-heavy models) against mobile data budgets (<100 MB/hour), favoring compression like sparsification (Yi et al., 2022). Latency for on-device inference must remain below 200 ms for top-$K$ recommendations on IoT-like devices (64 MB RAM) and 50-200 ms on cellphones (512 MB RAM), ensuring real-time responsiveness in non-IID settings (Zheng et al., 2024). These thresholds define "acceptable" deployment: e.g., peak memory <20% device total, energy <1 mJ per update, round time <10 s, validated empirically to sustain model utility without excessive churn (Lai et al., 2021; Qin et al., 2023). Comprehensive profiling across heterogeneous fleets quantifies trade-offs, guiding optimizations like participant selection or model pruning for production-scale federated movie recommenders (Damaskinos et al., 2022).

Dataset Description and Preprocessing

The raw MovieLens dataset (Lai et al., 2021), comprising explicit user ratings for movies, undergoes systematic preprocessing to align with the binary classification requirements of embedding-based federated recommender systems, addressing inherent challenges such as high sparsity (>95%), cold-start problems, and non-IID client distributions prevalent in real-world deployments (Ammad-ud-din et al., 2022; Wang et al., 2021). Specifically, the MovieLens 1M dataset, which includes ratings from 6,040 users on 3,900 movies, is frequently utilized to evaluate models focused on relational prediction tasks (Cai et al., 2024; Liang et al., 2021). For implicit feedback scenarios, these explicit ratings are often binarized to represent positive interactions, thereby transforming the problem into predicting user engagement rather than exact rating values (Minto et al., 2021). Furthermore, extensions like the HetRec-MovieLens dataset, which encompasses 860,000 ratings and integrates additional contextual information from sources like IMDb and Rotten Tomatoes, facilitate the exploration of richer feature spaces for classification tasks where ratings below a certain threshold (e.g., 3) are classified as negative instances (Wan et al., 2023). Common practice involves retaining users and items with a minimum number of interactions, typically 10 or 20, to mitigate extreme sparsity and cold-start effects, often employing a leave-one-out evaluation strategy where the latest interaction is reserved for testing (Nguyen et al., 2024; Yu et al., 2023). For instance, the MovieLens-1M dataset is commonly employed, where users who have rated at least 20 movies are included, and the dataset is partitioned into training and testing sets, often with a 6:2:2 ratio for train, validation, and test sets based on interaction timestamps per user (Na et al., 2025; Zhang et al., 2024). This binarization process converts ratings greater than three stars to positive interactions, while others are considered negative, thereby allowing for the generation of negative samples at a 1:4 ratio to balance the dataset for training robust recommendation models (Wang et al., 2021; Zheng et al., 2024). 

User and movie identifiers—typically arbitrary, sparse, and non-contiguous alphanumeric strings or large integers—are first remapped to dense, zero-indexed contiguous integer indices. This is accomplished by extracting unique user and movie IDs from the raw interaction records (user ID, movie ID, rating, timestamp), sorting them (e.g., by first occurrence or lexicographically), and constructing bijective dictionaries that assign sequential indices from $0 \text { to } N_{u}-1 \text { for users }$where $N_u$ denotes the number of unique users, and $0 \text { to } N_m-1 \text { for movies }$, where $N_m$ represents the number of unique movies. (Ammad-ud-din et al., 2022; Mancino et al., 2024; Zhang et al., 2024). This remapping optimizes memory access patterns and facilitates efficient lookup operations within embedding layers during model training and inference. Furthermore, explicit ratings, which typically range from 1 to 5 stars, are binarized into implicit feedback signals, where any rating above a predefined threshold (e.g., 3 or 4 stars) is converted to a positive interaction and all others are treated as negative, thus simplifying the learning objective to a presence/absence prediction. Specifically, ratings exceeding 3.5 are commonly binarized to 1, indicating a positive implicit preference, while ratings below this threshold are often considered as 0, signifying no explicit preference or a negative interaction, thus accommodating the typical distribution of user engagement (Cai et al., 2024). To mitigate data sparsity and cold-start issues, users and items with fewer than a predetermined number of interactions (e.g., five) are subsequently filtered out from the dataset (Zheng et al., 2024). This thresholding, often set to 5 interactions per user and 10 interactions per item, ensures a minimum level of user engagement and item popularity, preventing models from learning from extremely sparse or statistically insignificant data points (Wang et al., 2024). 

This standard practice, widely adopted in recommender system benchmarks and frameworks like Elliot (Anelli et al., 2021) and DataRec (Mancino et al., 2024), ensures full compatibility with embedding layers in deep learning frameworks (e.g., PyTorch's $\texttt{nn.Embedding}$ or TensorFlow's $\texttt{Embedding}$), where indices must be dense and start from zero to prevent embedding table wastage, enable efficient sparse-to-dense conversions, and optimize tensor operations such as gathering, indexing, and gradient scattering during local training and aggregation (Ammad-ud-din et al., 2022; Chen et al., 2021; Zhang et al., 2024). By maintaining one-to-one correspondences via invertible mappings, original interaction relationships are preserved without data loss or leakage risks, facilitating downstream tasks like adjacency matrix construction, batching for heterogeneous clients, and simulation of federated partitions (Lai et al., 2021; Wang et al., 2021). Additionally, this re-indexing permits the seamless integration of graph-based approaches, such as Graph Convolutional Networks, which can leverage these dense representations for enhanced recommendation accuracy and privacy in federated contexts (Peng et al., 2025). Furthermore, explicit ratings are binarized into implicit feedback signals to mitigate rating bias prevalent in explicit feedback systems, which is particularly challenging in federated settings where raw data is inaccessible (Gao et al., 2025). Specifically, ratings exceeding a predetermined threshold (e.g., 3.5 or 4 out of 5 stars) are transformed into implicit positive interactions (represented as 1), while others are treated as negative or unobserved interactions (Ammad-ud-din et al., 2022). This binarization process converts the original explicit feedback dataset into an implicit feedback matrix, aligning with common practices in federated recommendation systems that leverage implicit signals to reduce communication overhead and improve privacy (Minto et al., 2021). For instance, explicit ratings, such as 1-5-star ratings, are converted into binary values where 1 indicates an observed interaction and 0 represents an unobserved interaction (Wang et al., 2024). This transformation to implicit feedback, where an interaction is binary (1 if a user has engaged with an item, 0 otherwise), simplifies data handling and aligns with privacy-preserving techniques like local differential privacy, which often perform better with binary or count-based data (Minto, Haller, Haddadi, et al., 2021; Minto, Haller, Livshits, et al., 2021). This method allows for the representation of user-item interactions as a bipartite matrix where rows signify users and columns represent items, a common structure in recommendation systems that can be modeled using matrix factorization or graph-based methods (Fu et al., 2024). 

Ratings are binarized using a fixed global threshold of 4.0 on the original 5-point Likert scale, a standard practice in recommender systems research for MovieLens datasets (Dong et al., 2023; Mancino et al., 2024). Specifically, explicit ratings of 4 or 5 are converted to positive interactions (binary label 1), indicating strong user preference or liking for the movie, while ratings of 1, 2, or 3 are mapped to negative interactions (binary label 0), signifying dislike, neutrality, or indifference (Bernardis & Cremonesi, 2021; Dong et al., 2023; Pozo, 2016). This preprocessing step transforms the multi-class ordinal regression problem inherent in explicit ratings into a simplified binary classification task focused on preference prediction, aligning seamlessly with the requirements of embedding-based federated recommender systems that employ binary cross-entropy loss for efficient local training on sparse, non-IID client data (Ammad-ud-din et al., 2022; Dong et al., 2023; Mancino et al., 2024). This binarization reduces complexity for neural network models and ensures consistency across diverse client datasets, particularly crucial in federated learning where local model updates are aggregated (Ammad-ud-din et al., 2022; Wang et al., 2024). Further, categorical features like `Gender` and `Age` are converted into numerical representations, while `Genres` and `Titles` are transformed into dictionaries of numbers and then standardized in length for neural network processing (Zhang et al., 2024). These transformations prepare the heterogeneous raw data into a structured format suitable for direct input into embedding layers and subsequent neural network architectures, effectively mitigating sparsity and dimensionality issues (Zhang et al., 2024). Finally, the binarized ratings are implicitly interpreted as feedback, a technique commonly employed in federated collaborative filtering to simplify the task and enhance privacy (Minto et al., 2021). This binarization process converts the original explicit ratings into implicit feedback tuples, where $r_{ij} = 1$ denotes a positive interaction and $r_{ij} = 0$ a negative one, reflecting user engagement rather than subjective score, which is a common approach in recommender systems when dealing with large datasets (Zhang et al., 2025). 

The choice of 4.0 as the threshold reflects empirical consensus in the literature, as ratings ≥4 empirically capture "liked" items that users are likely to engage with positively, while lower scores do not, thereby preserving the core preference signal without the complexities of fine-grained rating prediction or user-specific normalization (Al-Ghossein, 2019; Dong et al., 2023; Shevchenko et al., 2024). This binarization mitigates rating scale biases across heterogeneous users, enhances compatibility with implicit feedback paradigms prevalent in collaborative filtering (e.g., via sampled softmax or BCE objectives (Mao et al., 2021; Yang et al., 2024)), and facilitates standardized evaluation using metrics like AUC, Recall@K, and Hit Rate@K in top-K recommendation scenarios (Shevchenko et al., 2024). In federated contexts, it further supports scalable model aggregation by standardizing interaction representations across clients, reducing variance from explicit rating discrepancies while addressing sparsity and cold-start challenges (Ammad-ud-din et al., 2022; Wang et al., 2021). This approach is consistent with common practices for dataset transformation where explicit feedback is converted into implicit feedback for training recommender systems (Wang et al., 2024; Zheng et al., 2024). To further simulate implicit feedback, all ratings are converted to a binary value, transforming them to 1, with negative instances sampled at a ratio of 1:4 (Minto et al., 2021; Zheng et al., 2024). This conversion facilitates the application of implicit feedback models, which are often more robust to data sparsity and can leverage a wider range of user interactions beyond explicit ratings (Minto et al., 2021; Polato, 2021). This approach is particularly beneficial in federated learning environments, where varying levels of user engagement and potential data discrepancies across edge devices necessitate a simplified yet effective representation of user preferences for robust model aggregation (Minto et al., 2021; Shevchenko et al., 2024). Furthermore, this binary representation, coupled with negative sampling, aligns with objective functions like binary cross-entropy, which are commonly employed in implicit feedback recommendation systems for training and model evaluation (Chen et al., 2025). For instance, a dataset like MovieLens, which initially contains explicit ratings, can be transformed into an implicit feedback dataset by binarizing interactions where a rating above a certain threshold (e.g., 4 out of 5) indicates a positive interaction, and then applying negative sampling to generate unobserved interactions as negative instances (Minto et al., 2021). This process involves considering only positive interactions and sampling non-interacted items as negative examples, usually at a 1:4 ratio, to ensure an appropriate balance between positive and negative signals during model training (Yuan et al., 2024). 

The preprocessed binary interactions are systematically partitioned into training and test sets employing a standard 80-20 temporal hold-out split on a per-user basis, a widely adopted protocol in recommender systems research to balance sufficient training data with realistic evaluation while mitigating cold-start issues for less active users (Anelli, Bellogín, et al., 2021; Anelli, Deldjoo, et al., 2021; Dacrema et al., 2021). Specifically, leveraging the timestamps available in the MovieLens dataset (Lai et al., 2021), interactions for each user are chronologically ordered, with the earliest 80% designated as the local training set—utilized for federated model optimization across simulated clients—and the most recent 20% reserved exclusively as the held-out test set for unbiased performance assessment (Bellogín & Said, 2021a, 2021b). This temporal separation emulates real-world deployment scenarios where recommendations are generated based on historical preferences to predict future engagements, thereby preventing data leakage that could arise from random splits incorporating future interactions into training and artificially inflating metrics such as NDCG or HR@K (Ji et al., 2022; Meng et al., 2020). This methodology also ensures that a clear distinction is maintained between observed data for training and unseen data for evaluation, which is critical for robustly assessing generalization capabilities in federated learning environments (Wei et al., 2024; Wu et al., 2023). This rigorous partitioning strategy, where the validation set is further derived from the training set using an 80/20 temporal hold-out split on a user basis, further optimizes learning rates and model parameters for the federated learning algorithms (Anelli et al., 2021). This approach not only provides a realistic evaluation of predictive performance but also addresses the challenge of non-IID data distributions inherent in federated learning by ensuring each client's local dataset reflects their specific temporal interaction patterns (Lim et al., 2025; Zhang et al., 2024). 

To ensure full reproducibility—a cornerstone of rigorous federated learning benchmarks amid statistical heterogeneity and non-IID distributions (Doriguzzi-Corin et al., 2025; Morafah et al., 2023)—a fixed random seed (e.g., 42) is enforced throughout the partitioning process, governing any stochastic elements like user ordering or minor random subsampling if applied for scalability (Rendle et al., 2019; Schmidt et al., 2024). This practice aligns with established frameworks such as Elliot (Anelli et al., 2021) and recommendations for standardized evaluation protocols in FL recsys (Ammad-ud-din et al., 2022; Wang et al., 2021), facilitating consistent client simulations (e.g., Dirichlet partitioning of training data into non-IID subsets (Li et al., 2022)) and cross-experiment comparability. By stratifying the split per-user, the approach preserves the inherent sparsity and heterogeneity of client data profiles, enabling robust assessments of federated aggregation under varying participation rates and privacy constraints (Jiang et al., 2024). Furthermore, to optimize hyperparameters and prevent overfitting, the training set is further subdivided into an 80/20 training/validation split, again on a user basis to maintain temporal ordering (Anelli et al., 2021). This subdivision ensures that model parameters are tuned on data distinct from the final test set, thereby providing a more robust evaluation of generalization performance (Zhang et al., 2024). This approach not only supports hyperparameter tuning and early stopping but also prevents the validation set from influencing the test set's evaluation, a critical consideration in preventing optimistic performance estimations (Ammad-ud-din et al., 2022; Tan et al., 2025; Yao et al., 2024). The careful partitioning of data into internal and external datasets, along with their subsequent division into training, validation, and test sets, is crucial for evaluating model performance and ensuring robust learning in federated settings (Xu et al., 2024). Subsequently, the partitioned training and test datasets are further divided into ten equal parts to simulate data distribution across ten distinct clients for federated learning, employing `np.array_split()` to distribute `X_train`, `y_train`, `X_test`, and `y_test` into respective storage lists (Gitanjali & Misra, 2024). This client-side data distribution enables the simulation of decentralized learning environments where each client independently trains a local model on its unique subset of data, reflecting the real-world scenarios of federated learning (Ali et al., 2025; Yang et al., 2024). This rigorous dataset partitioning strategy, including the maintenance of distinct validation sets per client, facilitates robust hyperparameter tuning and model performance evaluation across diverse client data distributions (Chen & Qi, 2025; Jozsa et al., 2025). 

Federated Learning Model Design

This section details the architectural choices and algorithmic considerations for the federated learning model, emphasizing how these designs address the specific challenges of mobile movie recommendation systems. Central to this is the careful selection of local model architectures, such as stacked autoencoders and hybrid collaborative filtering, which are trained on client-specific data before aggregation (Nivethitha & Aghila, 2024). The global model, an ensemble of decision trees, undergoes cost-complexity pruning at the server level to mitigate overfitting, a critical step for maintaining generalization across heterogeneous client data (Stephan et al., 2025). This approach allows for personalized model training on edge devices, where only model updates, not raw data, are transmitted to the central server for aggregation, ensuring privacy and reducing communication overhead (Sebastian et al., 2023). This framework also incorporates an autoencoder initialization strategy, ensuring a stable baseline for distributed model training and facilitating smoother convergence across diverse client datasets (Sánchez et al., 2024). The local models, trained on individual client data, transmit their structure, including split nodes, feature indices, threshold values, and Gini impurity criteria, to the central server for aggregation (Stephan et al., 2025). This aggregation process combines these individual model updates to form a more robust global model (Stephan et al., 2025). Specifically, a recursive technique, coined CFL, is employed for client selection and scheduling to account for incongruent data distributions, thereby creating specialized models which are then aggregated at the cloud level using both round-based and split-based schemes (Hamood et al., 2024). This hybrid aggregation approach not only enhances the overall model's predictive accuracy but also improves its adaptability to dynamic user preferences within the mobile movie recommendation ecosystem (Luo et al., 2022; Sebastian et al., 2023). This intricate process of local training and subsequent global aggregation, coupled with client selection strategies, allows the central server to form a more accurate and generalized predictor by combining these locally trained "weak" models (Hamood et al., 2024; Tsouparopoulos & Koutsopoulos, 2025). This federated paradigm inherently addresses privacy concerns by exchanging recommendation models rather than raw user data, a crucial aspect for sensitive applications like movie recommendations (Luo et al., 2022; Neumann et al., 2023). Furthermore, the architecture allows for individual client models to differ, potentially through variable pruning or custom designs, accommodating the resource constraints and unique data characteristics of diverse mobile devices (Desai et al., 2023). 

The server then collects and deserializes these local models, potentially employing weighted ensemble aggregation strategies rather than simple averaging, to construct a refined global model that optimizes overall performance (Khraisat et al., 2025). This server-side aggregation can leverage techniques like one-way hash mechanisms and stratified random sampling for verification during the aggregation phase, ensuring data integrity and facilitating recovery validation (Belhadi et al., 2024). To prevent local models from overfitting their training data, which can introduce noise during aggregation, the server subsequently resends all received local decision trees to the clients for evaluation against their respective private datasets (Argente-Garrido et al., 2024). This iterative process enables a continuous refinement of the global model while allowing clients to identify and rectify potential discrepancies or biases introduced during the aggregation, ultimately enhancing the overall model generalization and robustness (Tertulino, 2025). Such a federated learning architecture inherently addresses the challenges of data heterogeneity across clients, where non-IID data distributions are prevalent in real-world mobile recommendation systems (Duchesne et al., 2024; Jiang et al., 2024). Moreover, the paradigm's inherent privacy-preserving nature, by transmitting only model updates rather than raw data, significantly reduces the risks associated with centralized data collection, a critical concern for sensitive user information in recommendation systems (Neumann et al., 2023; Shen et al., 2020). This decentralized approach also supports scalability, as new devices can seamlessly join the federated network without requiring extensive infrastructure changes. 

Privacy Mechanisms Implemented

In this section, we delineate the specific privacy-preserving mechanisms integrated into our federated learning framework, which are crucial for safeguarding user data in mobile movie recommendation systems (Ammad-ud-din et al., 2019; Neumann et al., 2023; Prigent et al., 2024, p. 775). Our implementation adopts Differential Privacy (DP) as the primary privacy-preserving mechanism, specifically implementing the DP-Stochastic Gradient Descent (DP-SGD) algorithm adapted for federated learning environments. This approach provides formal privacy guarantees by injecting calibrated noise into the gradient computations during local client training, thereby preventing potential inference attacks that could extract sensitive information from model updates transmitted to the central server (Abadi et al., 2016; Wei et al., 2021). The introduction of such noise ensures that individual data points cannot be precisely reconstructed, even if an adversary gains access to the aggregated model updates (Luo et al., 2023, p. 11). This is particularly vital in recommender systems where user preferences, often considered sensitive, could otherwise be inferred from model parameters (Ammad-ud-din et al., 2019; Neumann et al., 2023). Furthermore, the system employs secure aggregation protocols to protect individual model updates from the server and other clients during the aggregation phase, ensuring that only the aggregated, noise-infused model is ever revealed (Yang et al., 2024, p. 4). This two-pronged approach, combining DP-SGD with secure aggregation, fortifies the privacy posture of the system against various adversarial threats, ensuring that user data remains protected throughout the distributed learning process while still contributing to a robust global model (Bonawitz et al., 2017; Marx et al., 2023; Xie et al., 2022). 

DP-SGD Implementation

The core privacy mechanism employed in our federated learning framework is DP-SGD, which modifies the standard stochastic gradient descent algorithm by incorporating two critical operations: gradient clipping and Gaussian noise addition (Du et al., 2021; McMahan et al., 2016). During each local training iteration on client devices, before model parameters are updated, the gradients computed through backpropagation are first clipped to bound their sensitivity, followed by the addition of calibrated Gaussian noise that scales proportionally with the clipping threshold. This two-step process transforms the standard gradient descent update into a differentially private mechanism that provides formal mathematical guarantees against privacy leakage, while maintaining the fundamental optimization dynamics required for effective model learning (Du et al., 2021; Pichapati et al., 2019).

Gradient Clipping Mechanism

The first stage of our DP-SGD implementation involves gradient clipping, which serves as a crucial sensitivity-bounding operation. For each training batch processed locally by a client, the gradient clipping operation computes the global L2 norm across all model parameter gradients by first flattening all gradient tensors from the model's parameter groups. This norm is then used to scale the gradients such that their L2 norm does not exceed a predefined threshold, thereby limiting the influence of any single data point or training batch on the overall model update (Shen et al., 2022; Wang et al., 2021). 

The choice of $C = 1.0$ as the clipping norm represents a careful balance between utility preservation and sensitivity bounding (Wu et al., 2023). A larger clipping threshold would allow gradients to retain more of their original magnitude, potentially preserving useful signal for model learning, but would require proportionally larger noise addition to maintain the same privacy guarantee (since noise scales with $C$), potentially degrading utility. Conversely, a smaller clipping threshold would reduce noise requirements but might excessively truncate informative gradients, leading to slower convergence or suboptimal model performance. Through empirical validation, we found that $C = 1.0$ provides an effective trade-off, allowing sufficient gradient flow for learning while maintaining manageable noise levels that preserve model utility across the privacy budget range we evaluated ($\epsilon \in \{∞, 8, 4, 2, 1\}$) (Biswas et al., 2025; Sander et al., 2022).

Gaussian Noise Addition

Following gradient clipping, calibrated Gaussian noise is added independently to each gradient component, drawn from a zero-mean multivariate Gaussian distribution with diagonal covariance matrix. Specifically, for each parameter tensor in the model, noise is sampled element-wise from $\mathcal{N}(0, (\sigma \cdot C)^2)$, where $\sigma$ is the noise multiplier (also referred to as the noise scale) and $C$ is the clipping norm. This noise addition process, mathematically expressed as 

$$\tilde{\nabla} \theta = \nabla \theta + \mathcal{N}(0, (\sigma \cdot C)^2 \mathbf{I})$$

ensures that the noisy gradient updates satisfy $(\epsilon, \delta)$- differential privacy, where $\epsilon$ quantifies the privacy budget consumed per training step and $\delta$ represents the probability of privacy failure, typically set to a small value such as $10^{-5}$ in our experiments (Abadi et al., 2016). The independence of noise across gradient components is crucial for maintaining the mathematical properties required for the privacy guarantee, while the per-parameter sampling ensures that the noise scale is appropriately calibrated to each gradient tensor's dimensionality and magnitude.

In our implementation, noise is added after the gradient clipping operation but before the optimizer step, ensuring that the noise is applied to the sensitivity-bounded gradients. The noise multiplier $\sigma$ is dynamically computed based on the target privacy budget $\epsilon$ using Renyi Differential Privacy. The relationship between $\sigma$ and the privacy guarantee depends on multiple factors including the number of training steps, batch size, sampling probability, and the RDP order parameter $\alpha$, making the RDP accountant essential for accurately determining the appropriate noise level for a given target privacy budget.

Privacy Budget Accounting

To accurately track and control the privacy budget consumption throughout the federated learning process, we implement the Rényi Differential Privacy accountant (Mironov, 2017a, 2017b). RDP, proposed as a natural relaxation of standard $(\epsilon, \delta)$-differential privacy based on the Rényi divergence of order $\alpha > 1$(Mironov, 2017a, 2017b), defines a mechanism $M$ as $(\alpha, \rho)$-RDP if $D_\alpha(M(D) || M) \leq \rho$ for neighboring datasets $D, D'$, where 

$$D_\alpha(P || Q) = \frac{1}{\alpha-1} \log \mathbb{E}_{o \sim Q} \left[ \left( \frac{P(o)}{Q(o)} \right)^\alpha \right]$$

captures tail bounds on privacy loss more precisely than pure $\epsilon$-DP(Mironov, 2017).

The RDP framework is particularly advantageous for tracking privacy loss in iterative algorithms like DP-SGD, as it enables simple additive composition: if $K$ identical mechanisms each satisfy $(\alpha, \rho)$-RDP, their composition satisfies $(\alpha, K\rho)$-RDP(Gopi et al., 2024; Mironov, 2017a, 2017b), vastly outperforming generic sequential composition theorems for $(\epsilon, \delta)$-DP that scale as $O(K \epsilon^2)$ or $O(\sqrt{K} \epsilon)$ under advanced composition(Asoodeh et al., 2020; Wang et al., 2020). This leads to smaller $\epsilon$ values for the same noise levels when converting accumulated RDP back to $(\epsilon, \delta)$-DP via

$$\epsilon = \min_{\alpha > 1} \left[ \rho(\alpha) + \frac{\log(\frac{1}{\delta})}{\alpha - 1} \right]$$

 (Mironov, 2017), or equivalently, less noise required to achieve a target privacy budget, thereby preserving greater model utility in resource-constrained federated settings(Asoodeh et al., 2020; Gopi et al., 2024).

RDP Computation for Gaussian Mechanism

Our RDP accountant computes the privacy budget for the Gaussian mechanism using the closed-form formula 

$$RDP_\alpha(\mathcal{M}) = \frac{\alpha} { 2\sigma^2}$$

where $\alpha$ is the Renyi order parameter (ranging over a discrete set of values from approximately 1.1 to 64 in our implementation, following standard practices) and $\sigma$ is the noise multiplier (Wang et al., 2025). This formula directly relates the noise level to the RDP guarantee, where larger noise (higher $\sigma$) results in smaller RDP values (better privacy), and vice versa. The RDP accountant then converts the accumulated RDP guarantee back to standard $(ε, δ)$-DP by optimizing over all possible values of $\alpha$ to find the minimum epsilon value that satisfies $ε = \min_\alpha [RDP_\alpha + \frac{\log(\frac{1}{δ})} { \alpha - 1}]$, where the minimization is performed over a predefined grid of $\alpha$ values to ensure computational tractability while maintaining accuracy. This RDP-based privacy accounting method allows for a more granular and accurate assessment of privacy leakage compared to traditional differential privacy accounting (Wang et al., 2025), particularly in the context of subsampled training inherent in federated learning environments (Ahmadi et al., 2025). We specifically utilized state-of-the-art differential privacy accountants such as Rényi Differential Privacy and FSRDP Accountant, which pre-compute the necessary noise for specified security parameters and data partition sizes. 

Composition Across Training Steps

The privacy accounting becomes more complex when composing the privacy loss across multiple training iterations, particularly in settings like DP-SGD where privacy guarantees must accumulate tightly over thousands of steps without excessive degradation of utility (Kairouz et al., 2021; Mironov, 2017). In our federated learning setup, each client performs local training over multiple batches within each federated round, and the total privacy cost accumulates across all batches and rounds, accounting for both intra-client subsampling (via batches) and inter-round client subsampling inherent to federated environments (Ahmadi et al., 2025, p. 3; Balle et al., 2020). For a client processing $T$ batches per round over $R$ federated rounds, the total number of training steps is $K = T \times R$. A key advantage of the Renyi Differential Privacy framework is its simple additive composition property: if each individual step (e.g., a subsampled Gaussian mechanism) satisfies $(\alpha, \rho)$-RDP for RDP order $\alpha > 1$, then the composition over $K$ identical steps satisfies $(\alpha, K \rho)$-RDP, where $\rho$ is the per-step RDP parameter (Feng et al., 2024, p. 6; Mironov, 2017; Wang et al., 2025, p. 6). This linear scaling vastly outperforms generic composition theorems for standard $(\epsilon, \delta)$-DP, enabling tighter overall bounds that are then converted back to $(\epsilon, \delta)$-DP via

$$\epsilon = \min_{\alpha > 1} \left[ K \rho(\alpha) + \frac{\log(\frac{1}{\delta})}{\alpha - 1} \right]$$

as detailed in the prior RDP computation section (Mironov, 2017; Toubeau et al., 2022, p. 804). Privacy amplification by subsampling further strengthens these guarantees: for batch sampling probability $q = B/N$ within a client and client sampling probability across rounds, the effective per-step RDP $\rho$ is reduced by factors depending on $q$, allowing less noise for the same privacy budget (Kairouz et al., 2021; Wang et al., 2022, p. 4). The number of steps per round is conservatively estimated as $T = \lceil N / B \rceil$ (or more generally $T = E \lceil N / B \rceil$ for $E$ local epochs), where $N$ is the average number of samples per client and $B$ is the batch size, ensuring the accounting accurately reflects the actual number of gradient computations and avoids underestimating the privacy loss (Li et al., 2024, p. 7; Pelikán et al., 2023, p. 47; Wei et al., 2024, p. 8).

Adaptive Noise Calibration

In our experimental setup, we evaluate the privacy-utility trade-off across a spectrum of privacy budget levels, specifically testing $\epsilon \in \{\infty, 8, 4, 2, 1\}$, where $\epsilon = \infty$ corresponds to the non-private baseline without any differential privacy protection and lower $\epsilon$ values indicate progressively stronger privacy guarantees at the cost of increased noise and potential utility degradation(Li et al., 2023, p. 103155; Wu et al., 2017, p. 17; Zhou et al., 2020, p. 10). These logarithmically spaced $\epsilon$ values are standard in the differential privacy machine learning literature, enabling comprehensive assessment of the sensitivity of model performance to varying privacy strengths across weak to strong regimes(Jayaraman & Evans, 2019).

For each target $\epsilon$ (with fixed $\delta = 10^{-5}$), the RDP accountant employs an efficient binary search algorithm over the noise multiplier $\sigma$ space to determine the minimal $\sigma$ that achieves the desired privacy guarantee while minimizing excess noise(Fu et al., 2024, p. 1202; Saifullah et al., 2023, p. 20; Yang, 2025, p. 6). This search exploits the monotonicity of the privacy loss with respect to $\sigma$: larger $\sigma$ reduces the computed $\epsilon$ by amplifying Gaussian noise relative to the clipped gradient sensitivity(Tantipongpipat et al., 2021, p. 6; Zhang & He, 2023, p. 6).

The binary search procedure initializes bounds in the range $[\log \sigma_{\min} = -1, \log \sigma_{\max} = 1]$ (typically $\sigma \in [0.1, 10]$ in our implementation to cover practical regimes observed in DP-SGD benchmarks(Zhou et al., 2020, p. 10)), iteratively tests candidate $\sigma_{\text{mid}}$ values, computes the resulting $\epsilon_{\text{computed}}$ via RDP accounting—summing per-step $(\alpha, \rho)$-RDP over $K = T \times R$ steps (with subsampling $q$), minimizing over $\alpha \in \{2, 4, \dots, 128\}$, and converting to $(\epsilon, \delta)$-DP(Mironov, 2017; Yang, 2025, p. 6)—and narrows the search space: if $\epsilon_{\text{computed}} > \epsilon_{\text{target}}$, set $\sigma_{\text{low}} \leftarrow \sigma_{\text{mid}}$; otherwise, $\sigma_{\text{high}} \leftarrow \sigma_{\text{mid}}$. The search terminates when $|\epsilon_{\text{computed}} - \epsilon_{\text{target}}| \leq 0.01$ (a common tolerance for numerical stability) or after a maximum of 50 iterations to bound computation time(Saifullah et al., 2023, p. 20). This adaptive calibration ensures that the optimal noise scale is applied to achieve the target privacy budget without unnecessarily compromising model utility (Cyffers & Bellet, 2020, p. 8; Saifullah et al., 2024, p. 494). 

This adaptive noise calibration process incorporates several key hyperparameters that govern the Rényi Differential Privacy accounting and noise multiplier $\sigma$ optimization in DP-SGD for federated learning: the number of federated rounds $R$, the average number of samples per client $N$, the local batch size $B$, and the per-sample gradient clipping norm $C=1.0$(Morsbach et al., 2024, p. 2; Pelikán et al., 2023, p. 47; Pfitzner & Arnrich, 2022, p. 3). Specifically, these determine the intra-client subsampling rate $q = B/N$, the batches per client per round $T = \lceil N/B \rceil$ (or $T = E \lceil N/B \rceil$ for $E$ local epochs), and the total composition steps $K = R \times T$, which directly scale the cumulative RDP parameter $\rho_{\text{tot}}^\alpha = K \cdot \rho^\alpha(q, \sigma/C)$ across $\alpha \in \{2, 4, \dots, 128\}$(Ling et al., 2024, p. 355; Wei et al., 2024, p. 8; Yang, 2025, p. 6). The clipping norm $C$ bounds gradient sensitivity to $C$, enabling Gaussian noise calibration as $\mathcal{N}(0, \sigma^2 C^2 \mathbf{I})$ post-clipping, while larger $N$ or smaller $q$ amplifies privacy via subsampling(Andrew et al., 2019; Wang et al., 2025, p. 6).

A binary search over $\sigma$ (e.g., initial bounds $[0.1, 10]$) iteratively computes the resulting

$$\epsilon_{\text{computed}} = \min_\alpha \left[ \rho_{\text{tot}}^\alpha + \frac{\log(\frac{1}{\delta})}{\alpha-1} \right]$$

until $|\epsilon_{\text{computed}} - \epsilon_{\text{target}}| \leq 0.01$, yielding the minimal $\sigma$ that tightly meets the target $(\epsilon, \delta)$-DP budget (e.g., $\delta=10^{-5}$)(Morsbach et al., 2024, p. 2; Yang, 2025, p. 6). This dynamic adjustment via the RDP accountant—exploiting its additive composition and tight conversion to $(\epsilon, \delta)$-DP—prevents privacy under-protection (excessive leakage) or over-protection (unnecessary utility loss from surplus noise), outperforming generic composition(Toubeau et al., 2022, p. 804; Wang et al., 2025, p. 6; Wei et al., 2021, p. 805). Calibration occurs once prior to training for each experimental setup (e.g., per $\epsilon \in \{\infty, 8, 4, 2, 1\}$), with the fixed $\sigma$ applied uniformly across all subsequent rounds to maintain consistent guarantees without runtime overhead(Ahmadi et al., 2025, p. 6; Pfitzner & Arnrich, 2022, p. 3; Saifullah et al., 2024, p. 494).

Integration with Federated Learning

This fixed noise scaling parameter $\sigma$ is then incorporated into the federated learning training loop, where it directly influences the magnitude of the Gaussian noise added to the aggregated local model updates from clients, ensuring that the desired $(\epsilon, \delta)$- differential privacy guarantee is upheld throughout the iterative model optimization process (Ranaweera et al., 2025, p. 9; Wei et al., 2021, p. 801). 

The DP-SGD mechanism is seamlessly integrated into the federated learning training loop at the client level, where each participating device receives the current global model from the server at the start of each round, initializes its local model accordingly, and performs multiple local epochs or steps of stochastic gradient descent on its private data.(Riedel et al., 2023, p. 209; Xia et al., 2025, p. 3) During this local training phase, for each minibatch drawn from the client's dataset (typically via subsampling with rate $q = B/N$), the client computes per-sample gradients via backpropagation, clips their $\ell_2$-norms to a bound $C$ (e.g., per-sample or per-client clipping), averages the clipped gradients, and adds calibrated Gaussian noise $\mathcal{N}(0, \sigma^2 C^2 I)$ calibrated via the RDP accountant to achieve the target $(\epsilon, \delta)$-DP guarantee.(Andrew et al., 2019; Hou et al., 2023, p. 1; Weng et al., 2022, p. 4; Xia et al., 2025, p. 3) The optimizer then updates the local model parameters using these noisy, clipped gradients, repeating this process for $T = E \lceil N/B \rceil$ steps per round before transmitting the final local model update to the central server.(Pelikán et al., 2023, p. 47)

This client-side privacy protection, akin to local differential privacy, ensures that even if the central server is honest-but-curious, communication channels are intercepted, or the server is compromised, the raw or unperturbed gradient information remains obfuscated, as noise is injected at the source prior to any transmission.(Ahmadi et al., 2025, p. 2; Islamov et al., 2025, p. 4; Stojkovic et al., 2022; Wei et al., 2021; Weng et al., 2022, p. 4) By distributing clipping and noise addition across devices, it provides defense-in-depth privacy guarantees that protect user data at multiple layers—from local computation to aggregation—preventing gradient leakage attacks and eliminating exposure of individual contributions to a potentially untrusted server, unlike server-side approaches such as DP-FedAvg which add noise only post-aggregation.(Hou et al., 2023, p. 1; Ranaweera et al., 2025, p. 4; Wei & Liu, 2024, p. 19) This decentralized application aligns with federated learning's data minimization principle, enhancing robustness while leveraging privacy amplification by subsampling.(Kairouz et al., 2021; Wang et al., 2022, p. 4)

Per-Batch Privacy Protection

During each federated learning round, the privacy protection process in DP-SGD operates at the granular level of individual training batches (or minibatches) processed locally on each client device, ensuring per-step differential privacy amplification through subsampling and noise injection(Andrew et al., 2019; Xia et al., 2025, p. 3). At the outset of the round, selected clients receive the current global model parameters $\theta^{t-1}$ from the central server, initialize their local model copy $\theta^{t,0}_i \leftarrow \theta^{t-1}$, and proceed to perform $T = E \lceil N_i / B \rceil$ local training steps (where $E$ is the number of local epochs, $N_i$ the client's dataset size, and $B$ the minibatch size), iterating over their private local data $\mathcal{D}_i$(Pelikán et al., 2023, p. 47; Riedel et al., 2023, p. 209).

For each local step $t_{\text{loc}} \in \{1, \dots, T\}$:





Minibatch subsampling: A minibatch $\tilde{\mathcal{B}}^{t,t_{\text{loc}}}_i \subset \mathcal{D}_i$ of size $B$ is randomly sampled (e.g., with or without replacement, or Poisson sampling), yielding an effective subsampling rate $q = B / N_i$ that amplifies privacy via the subsampling lemma in the RDP framework(Kairouz et al., 2021; Xia et al., 2025, p. 3).



Forward pass: The local model computes predictions $\hat{y}_j = f(\theta^{t,t_{\text{loc}}-1}_i; x_j)$ for each sample $(x_j, y_j) \in \tilde{\mathcal{B}}^{t,t_{\text{loc}}}_i$



Per-sample loss and gradients: Individual losses $\ell(\hat{y}_j, y_j)$ are computed using the mean squared error criterion, $\ell = \frac{1}{B} \sum_j \| \hat{y}_j - y_j \|_2^2$; backpropagation then yields per-sample gradients $g_j = \nabla_\theta \ell(\hat{y}_j, y_j)$ for each $j$ (facilitated in PyTorch via libraries like Opacus, which use gradient hooks or micro-batches to isolate per-sample contributions without full recomputation)(Fares & Sertbaş, 2024, p. 5).



Per-sample gradient clipping: Each per-sample gradient is clipped to bound its $\ell_2$-norm sensitivity: $\bar{g}_j = g_j \cdot \min\left(1, \frac{C}{\|g_j\|_2}\right)$, where $C=1.0$ is the fixed clipping norm, preventing any single sample from dominating the update(Andrew et al., 2019; Pelikán et al., 2023, p. 20; Xia et al., 2025, p. 3).



Average and noise injection: The average clipped gradient is formed as $\bar{g}^{t,t_{\text{loc}}}_i = \frac{1}{B} \sum_j \bar{g}_j$, to which calibrated isotropic Gaussian noise is added: $\tilde{g}^{t,t_{\text{loc}}}_i = \bar{g}^{t,t_{\text{loc}}}_i + \mathcal{N}(0, \sigma^2 C^2 \mathbf{I})$, with noise multiplier $\sigma$ pre-calibrated via binary search on the RDP accountant to meet the target $(\epsilon, \delta)$-DP budget across all $K = R \times T$ steps(Xia et al., 2025, p. 3; Yang, 2025, p. 6).



Optimizer update: The noisy gradient $\tilde{g}^{t,t_{\text{loc}}}_i$ drives the local optimizer step, e.g., $\theta^{t,t_{\text{loc}}}_i \leftarrow \theta^{t,t_{\text{loc}}-1}_i - \eta \tilde{g}^{t,t_{\text{loc}}}_i$(using Adam or SGD), repeating for all local steps before the final client update $\Delta^t_i = \theta^{t,T}_i - \theta^{t,0}_i$ is transmitted(Hou et al., 2023, p. 1; Pelikán et al., 2023, p. 47).

This per-batch (per-step) DP-SGD application—distinct from coarser client-level clipping of the full local update—ensures tight privacy accounting under advanced composition (additive RDP $\rho^\alpha_{\text{tot}} = K \cdot \rho^\alpha(q, \sigma)$ over orders $\alpha \in \{2,\dots,128\}$), uniform protection across computations, and robustness to outliers via per-sample bounding, while leveraging subsampling for enhanced utility without runtime overhead(Andrew et al., 2019; Kairouz et al., 2021; Yang, 2025, p. 6).

Client-Side Privacy vs. Server-Side Privacy

An important design choice in our implementation is the application of DP-SGD at the client level rather than at the server level after aggregation. Client-side privacy protection provides several advantages: first, it ensures that even if model updates are intercepted during transmission or if the server is compromised, the raw gradient information remains protected; second, it distributes the computational overhead of noise generation across all clients rather than concentrating it at the server; and third, it prevents the server from ever observing unperturbed gradients, eliminating potential attack vectors that could exploit gradient information before aggregation.This approach inherently aligns with the principles of federated learning, where sensitive user data remains decentralized on devices, minimizing central points of failure and enhancing overall system security (Stojkovic et al., 2022). Conversely, server-side noise addition, as seen in DP-FedAvg, provides computational efficiency by adding noise only once during aggregation, thereby protecting the global model rather than individual client data (Ranaweera et al., 2025). However, this approach requires careful consideration of the privacy composition, as each client's local training involves multiple batches, and the privacy cost accumulates across all batches processed by all clients throughout the federated learning process. This decentralized application of differential privacy aligns with the principle of data minimization, ensuring that sensitive user data never leaves the device in an unmasked form (McMahan et al., 2016; Stojkovic et al., 2022). This contrasts with server-side DP mechanisms, where noise is applied after aggregation, potentially exposing individual contributions to an honest-but-curious server before perturbation (Commey et al., 2024; Weng et al., 2022). However, some approaches, like DP-FedAvg, add noise during the aggregation of model updates from each client, providing strong privacy guarantees at both local and global model levels while potentially enhancing computational efficiency by injecting noise only once (Ranaweera et al., 2025). Our chosen methodology, by injecting noise at the client level during local training, offers robust protection against gradient leakage attacks that could occur before model sanitization (Wei & Liu, 2024). This design prioritizes safeguarding individual data contributions at their origin, ensuring that the privacy budget is consumed locally before any information is shared (Thapa et al., 2022). This approach provides a stronger privacy guarantee against a potentially untrusted server, as it never receives unperturbed gradients (Ahmadi et al., 2025). This client-level DP application aligns with the concept of local differential privacy, which offers stronger protection against a semi-honest server compared to central differential privacy models (Islamov et al., 2025). 

Aggregation of Noisy Updates

After clients complete their local training with DP-SGD, they transmit their updated model parameters (which have been trained using clipped and noisy gradients) back to the server for aggregation. The server aggregates these noisy updates using standard federated averaging (weighted by the number of samples per client), computing, 

$$\theta_{global}^{(t+1)} = \sum_{i=1}^{n} \frac{n_i}{N} \theta_i^{(t+1)}$$

where $\theta_i^{(t+1)}$ is the locally updated model from client $i$ (trained with DP-SGD), $n_i$ is the number of training samples at client $i$, and $N = \sum_i n_i$ is the total number of samples (Ranaweera et al., 2025).  Since each client's update has already been protected by DP-SGD during local training, the aggregated global model inherits the privacy guarantees from the individual client updates. The privacy accounting tracks the cumulative privacy cost across all training rounds, ensuring that the total privacy budget consumption does not exceed the predefined epsilon limit, with the RDP accountant computing the final privacy guarantee at the conclusion of training.

Privacy Parameter Configuration

Our implementation maintains a fixed $\delta = 10^{-5}$ across all experiments, representing the probability that the $(\epsilon, \delta)$-differential privacy guarantee fails—a small additive relaxation allowing tighter privacy accounting with Gaussian mechanisms compared to pure $\epsilon$-DP(Gomez et al., 2025; Vinterbo, 2022, p. 1). This choice aligns with standard practices in the differential privacy literature, where $\delta$ is typically set smaller than $\frac{1}{n}$ (with $n$ the dataset size) to ensure meaningful protection against privacy failure events exceeding the inverse dataset scale, often on the order of $10^{-5}$ to $10^{-6}$ or cryptographically negligible values(Dibia et al., 2025, p. 7; Gomez et al., 2025, p. 11; Lin, 2022, p. 15). For the MovieLens dataset with approximately 6,706 training samples ($\frac{1}{n} \text{approx } 1.5 \times 10^{-4}), \delta = 10^{-5}$ comfortably satisfies this heuristic, rendering failure probability negligible relative to dataset scale while enabling advanced composition via the RDP accountant without undue utility loss(Vinterbo, 2022, p. 1). Fixing $\delta$ independent of varying $\epsilon$ budgets simplifies hyperparameter tuning and ensures consistent privacy failure risk across regimes, as $\epsilon$ primarily controls distinguishability while $\delta$ bounds tail events(Bu et al., 2020, p. 14).

The per-sample gradient clipping norm is fixed at $C = 1.0$ across all privacy budgets, a common empirical default in DP-SGD that bounds l2-sensitivity to $\Delta = C$, calibrating Gaussian noise $\sigma \propto C / \epsilon$ via the RDP accountant(Annamalai & Cristofaro, 2024, p. 3; Huang et al., 2024, p. 1). This value strikes a practical balance: sufficiently large to preserve typical gradient magnitudes in neural networks (avoiding excessive clipping bias that degrades convergence(Koloskova et al., 2023; Shulgin & Richtárik, 2024, p. 1)) yet tight enough to minimize noise variance for utility, especially under heavy-tailed gradients or federated heterogeneity(Andrew et al., 2019; Das et al., 2022). Fixed $C$ simplifies deployment over adaptive schemes (which estimate quantiles online at minor privacy cost(Andrew et al., 2019; Bu et al., 2022)), standardizing sensitivity across $\epsilon$ levels and model scales while empirically mitigating outliers without task-specific retuning(Bu et al., 2022; Lin, 2022, p. 15). Sensitivity analyses confirm $C=1.0$ incurs minimal utility degradation (e.g., <2% RMSE increase in recommendation tasks(Agrawal et al., 2023, p. 7; Wu et al., 2022, p. 5)), aligning with per-batch DP-SGD's robustness in the preceding local training protocol.

Privacy-Utility Trade-off Observations

Experimental results confirm that our client-side DP-SGD configuration with fixed $\delta=10^{-5}$ and $C=1.0$ achieves the target $(\epsilon, \delta)$-differential privacy guarantees across all tested budgets while preserving acceptable model utility for personalized movie recommendations(Dwork et al., 2014). As anticipated from differential privacy theory, where lower $\epsilon$ demands proportionally larger Gaussian noise $\sigma \propto C / \epsilon$ to bound the privacy loss, the privacy-utility trade-off intensifies with decreasing $\epsilon$, perturbing per-sample gradients more aggressively and introducing greater stochasticity in local updates(Andrew et al., 2019; Huang et al., 2024, p. 1). This noise amplification hinders convergence by increasing gradient variance, particularly under federated data heterogeneity, yet advanced RDP accounting enables tight budget tracking for practical regimes(Vinterbo, 2022, p. 1).



At $\epsilon = 8$, noise levels remain modest, yielding model performance proximate to the non-private baseline ($\epsilon = \infty$). Recommendation metrics exhibit marginal degradation: $Hit@10$ (Hit Rate at 10, the proportion of ground-truth positives in the top-10 predicted items) and $NDCG@10$ (Normalized Discounted Cumulative Gain at 10, rewarding relevant higher-ranked recommendations) drop by less than 5-10% relative to baseline, akin to observations in DP-enhanced federated recommenders on MovieLens(Minto et al., 2021, p. 1; Wang et al., 2021, p. 889; Wu et al., 2022, p. 9). Progressing to $\epsilon = 4, 2,$ and $1$, the noise multiplier escalates substantially—often by factors of 2-8x—exacerbating impacts on training stability, slowing global model convergence across federated rounds, and elevating final test errors(Imran et al., 2022, p. 19; Shiva & Isabel, 2025; Wang et al., 2021, p. 890).

Nevertheless, at $\epsilon = 1$—affording strong protection where an adversary's distinguishability between adjacent user datasets is bounded by roughly $e^1 \approx 2.7$—the model upholds meaningful utility, with $Hit@10 > 0.5$ and competitive $NDCG@10$ feasible even on full-scale MovieLens, mirroring empirical successes in local DP-FL for implicit feedback(Minto, Haller, Haddadi, et al., 2021, p. 1; Minto, Haller, Livshits, et al., 2021, p. 2). Such retention validates the practical deployability of privacy-preserving federated learning in resource-constrained mobile recommendation systems, where user-level protections outweigh moderate utility costs(“A Privacy-Preserving Federated Recommender System with Neuro-Fuzzy Modeling and Local Differential Privacy,” 2025; Chen et al., 2022, p. 21). This trade-off is rigorously quantified in the experimental results section through sweeps of $\epsilon$ against multiple metrics (e.g., $Hit@K$, $NDCG@K$, RMSE), alongside ablation on noise sensitivity and comparison to server-side DP baselines.

Privacy Guarantee Formalization

The differential privacy guarantee provided by our implementation can be formally stated as follows: for any two adjacent datasets $D$ and $D'$ that differ in at most one training sample, the probability distribution of the output model parameters (or gradients) satisfies 

$$Pr[\mathcal{M}(D) \in S] \leq e^\epsilon \cdot Pr[\mathcal{M}(D') \in S] + \delta$$

for any subset $S$ of possible outputs, where $\mathcal{M}$ represents the DP-SGD mechanism. This definition captures the fundamental privacy guarantee: the presence or absence of any single individual's data in the training set should not significantly change the probability distribution of the mechanism's output, making it difficult for an adversary to infer membership in the training set or extract sensitive information about specific users. This formalization rigorously establishes that the DP-SGD mechanism effectively prevents reconstruction attacks, where an adversary attempts to infer individual-specific data from aggregated model updates, even under strong adversary models (Badidi et al., 2025; Pham et al., 2023). Moreover, local differential privacy is also integrated into some federated learning frameworks to enhance transparency and mitigate gradient leakage attacks by injecting Laplace noise into locally trained model gradients before transmission to the server (“A Privacy-Preserving Federated Recommender System with Neuro-Fuzzy Modeling and Local Differential Privacy,” 2025; Xu et al., 2025). 

Privacy Against Inference Attacks

In the context of federated learning, the $(\epsilon, \delta)$-differential privacy guarantee ensures that an adversary observing the noisy model updates transmitted from clients to the server cannot determine with high confidence whether any specific user's full interaction data was included in the local training process. This provides robust protection against membership inference attacks, where a malicious server or curious client attempts to ascertain whether a particular user's dataset—such as their complete movie viewing history on a platform like MovieLens—contributed to the global model(Wang et al., 2021, p. 879; Yuan et al., 2023a, 2023b, p. 1053). Formally, for adjacent user-level datasets $D$ and $D'$ differing only in one user's data, the output distributions satisfy $\Pr[\mathcal{M}(D) \in S] \leq e^\epsilon \Pr[\mathcal{M} \in S] + \delta$ for any measurable output set $S$, rendering individual contributions statistically indistinguishable and thwarting MIAs even under strong adversary models with auxiliary knowledge of user behaviors(Minto et al., 2021; Zhang et al., 2023, p. 987).

The DP guarantee extends to attribute inference attacks, where adversaries exploit model updates or outputs to infer sensitive user attributes, such as age, gender, demographics, or inferred lifestyle traits from interaction patterns. In FL recommender systems, vanilla setups are vulnerable as gradient updates encode user-specific embeddings reflective of private attributes; however, DP-SGD's per-sample clipping and Gaussian noise injection masks these signals, with empirical studies showing significant reductions in attacker F1-scores (e.g., >50% drop under $\epsilon \approx 1-10$) without substantial utility loss on datasets like MovieLens(Wang et al., 2021, p. 879; Yuan et al., 2023, p. 1053; Zhang et al., 2023, p. 994). Adaptive DP variants further optimize by allocating tighter budgets to vulnerable model components, like user embeddings, enhancing resistance while preserving recommendation accuracy(Zhang et al., 2023, p. 992).

Additionally, the calibrated Gaussian noise ($\sigma \propto C / \epsilon$) in the DP-SGD mechanism provides strong defense against gradient inversion attacks or data reconstruction attacks, where adversaries reverse-engineer individual training samples (e.g., specific movie ratings or implicit feedback) from observed aggregated gradients. By obscuring the true gradient geometry and magnitudes, noise prevents high-fidelity reconstructions, as demonstrated in FL settings where DP noise multipliers ($\sigma > 0.3$) render inverted images or interactions no more accurate than random priors, even for "high-risk" clients with sparse data(Hatamizadeh et al., 2023, p. 2050; Neumann et al., 2023; You et al., 2024). This is complemented by early client-side aggregation in some schemes, further diluting per-user signals.

This robust privacy framework is crucial for safeguarding sensitive user data in mobile movie recommendation systems, where personal viewing habits and preferences—revealing tastes in genres, emotional states, or social influences—could otherwise be vulnerable to sophisticated attacks like those leveraging model inversion or poisoning(Chen et al., 2022, p. 21; Minto et al., 2021; Neumann et al., 2023). Such protection is particularly vital for mitigating risks associated with the exploitation of personalized user data, including driving patterns, location histories, and behavioral traits (Badidi et al., 2025, p. 5). This rigorous approach ensures that individual privacy is preserved while still enabling the collaborative benefits of federated learning for enhancing recommendation accuracy, achieving viable metrics like $Hit@10 > 0.5$ under strong DP regimes ($\epsilon \leq 1$) on full-scale MovieLens(Banse et al., 2024, p. 1; Minto et al., 2021a, p. 1, 2021b; Shen et al., 2022, p. 3; Wang et al., 2021, p. 879). This ensures that the widespread adoption of federated learning in sensitive domains like mobile movie recommendation systems aligns with ethical data practices and regulatory compliance (Chilukoti et al., 2025, p. 101479; Jimenez-Gutierrez et al., 2025, p. 25). 

Compositional Privacy Guarantees

The privacy guarantees of differential privacy mechanisms, such as those provided by DP-SGD, hold compositionally across multiple training rounds and heterogeneous client participations in federated learning(Feng et al., 2024, p. 3; Wang et al., 2021, p. 884). This means that the cumulative privacy loss from sequentially applying the mechanism—once per round or per client update—can be precisely tracked and bounded, ensuring the overall process does not exceed a predefined total privacy budget $(\epsilon, \delta)$. In naive sequential composition, the privacy parameters add up linearly: if each of $T$ rounds satisfies $(\epsilon_t, \delta_t)$-DP, the composition yields $\left(\sum \epsilon_t, \sum \delta_t\right)$-DP(Wang et al., 2021, p. 894; Zhang et al., 2024, p. 3). However, this pessimistic bound quickly exhausts the budget in iterative settings like federated learning, where $T$ can reach hundreds or thousands, severely degrading utility due to excessive noise(Toubeau et al., 2022, p. 804).

To address this, the Rényi Differential Privacy framework, introduced by Mironov(Mironov, 2017), provides tight composition bounds via simple addition of RDP parameters: if mechanisms $M_1, \dots, M_T$ each satisfy $(\alpha, \epsilon_i(\alpha))$-RDP for Rényi order $\alpha > 1$, their composition satisfies $(\alpha, \sum \epsilon_i(\alpha))$-RDP(Wang et al., 2022, p. 4; Wei et al., 2024, p. 5; Yuan et al., 2024, p. 4). RDP is then converted to standard $(\epsilon, \delta)$-DP using $\epsilon = \min_\alpha \left[ \epsilon(\alpha) + \frac{\log(1/\delta)}{\alpha-1} \right] $(Feng et al., 2024, p. 3; Wang et al., 2021, p. 890). The RDP accountant (or moments accountant(Wang et al., 2021, p. 884)) tracks this cumulative loss efficiently by maintaining the Rényi divergence orders symbolically or numerically, enabling analytical privacy amplification from subsampling (common in FL client selection) and yielding up to 2-10x tighter budgets than sequential composition(Behnia et al., 2025, p. 7; Sun et al., 2024, p. 13; Toubeau et al., 2022, p. 804).

In DP-SGD for federated learning, per-round privacy costs arise from Gaussian noise addition after per-sample clipping, with noise $\sigma \propto C / \epsilon$ (clip norm $C$). The accountant computes step-wise RDP as $\epsilon(\alpha) = \frac{\Delta^2 \alpha}{2\sigma^2}$ for Gaussian mechanisms ($\Delta$: sensitivity), amplified by sampling rate $q$ via $\epsilon'(\alpha) \approx O\left( \frac{\alpha q^2}{2\sigma^2} \right)$(Wang et al., 2022, p. 4; Wei et al., 2024, p. 5). This is crucial for FL recommender systems, where training spans many rounds on sparse user data (e.g., MovieLens interactions): the RDP accountant's optimized analysis allows models to train over 10+ federated rounds (or even 100s with amplification(Asoodeh et al., 2020)) at $\epsilon \leq 1$, $\delta < 10^{-5}$, achieving viable Hit@10 $>0.5$ without linear budget explosion(Kairouz et al., 2021; Wang et al., 2021, p. 890; Zhang et al., 2024, p. 3). Empirical studies confirm: on benchmarks, RDP enables 2-5x more iterations than basic composition before budget exhaustion, balancing privacy against utility in real-world deployment(Behnia et al., 2025, p. 7; Yang, 2025, p. 6).

Furthermore, RDP supports advanced features like privacy amplification by iteration in contractive updates(Feldman et al., 2018) and heterogeneous budgets across clients(Lin & Wang, 2025, p. 2), vital for mobile recommenders with varying participation. Post-processing invariance ensures server aggregation or adaptive optimizers (e.g., DP-Adam) incur no extra loss(Aguilera-Martínez & Berzal, 2025, p. 10).

Furthermore, the post-processing invariance property of differential privacy ensures that any deterministic or randomized computation applied to a differentially private output—such as server-side aggregation of client gradients, adaptive optimizers (e.g., Adam or RMSProp), or model selection—preserves the original $(\epsilon, \delta)$-DP guarantees without incurring additional privacy expenditure (Aguilera-Martínez & Berzal, 2025, p. 10; Li et al., 2022, p. 2; Wang et al., 2021, p. 894; Xie et al., 2023, p. 1517). This is particularly advantageous in federated learning settings for mobile movie recommenders, where noisy per-client gradients $\tilde{g}_i^t = g_i^t + \mathcal{N}(0, \sigma^2 I)$ (with $\sigma \propto C / \epsilon$, $C$ the clip norm) are securely averaged at the server as $\tilde{G}^t = \frac{1}{K} \sum_{i=1}^K \tilde{g}_i^t$, followed by further processing like momentum updates or projection steps, all without amplifying the cumulative privacy loss tracked via RDP accountants (Feldman et al., 2018; Kairouz et al., 2021). Such flexibility enables seamless integration of downstream tasks, like hyperparameter tuning or ensemble methods, while maintaining rigorous user-level privacy bounds.

This robust privacy framework thus significantly bolsters the security posture of mobile movie recommendation systems against diverse inference threats, including membership inference and attribute inference attacks from curious clients—who may collude to probe global model updates for user participation or profile reconstruction—and honest-but-curious servers seeking to reverse-engineer sensitive viewing histories from aggregated gradients (Feng et al., 2024, p. 2; Minto et al., 2021; Yuan et al., 2023, p. 1053; Zhang et al., 2023, p. 991). Empirical defenses confirm that DP noise ($\sigma > 0.3$) effectively masks these signals, rendering inferences no better than random guessing even under sparse interaction data typical of mobile users (Hatamizadeh et al., 2023, p. 2050; Neumann et al., 2023).

However, the efficacy of differential privacy hinges on meticulous noise calibration, as suboptimal $\sigma$ or clip norms $C$—often fixed across heterogeneous clients—can precipitate substantial utility degradation, particularly over successive training iterations where composition exhausts the privacy budget via linear or moments accountant accumulation (Kairouz et al., 2021; Piran et al., 2025, p. 4; Wei et al., 2024, p. 5; Zhang et al., 2024, p. 3). In iterative DP-SGD for federated recommenders, excessive early-round noise stifles convergence on sparse MovieLens data, dropping Hit@10 below viable thresholds ($<0.5$), while adaptive schemes dynamically tuning $\sigma^t \propto C^t q / \epsilon$ (with sampling rate $q$) via per-layer clipping or convergence-aware scaling mitigate this, enabling 2-5x more rounds at $\epsilon \leq 1$ without linear utility erosion (Andrew et al., 2019; Du et al., 2021; Wang et al., 2021, p. 884).

Adjacent Dataset Definition

In our federated learning context for mobile movie recommendation systems, two datasets $D$ and $D'$ are formally considered adjacent (or neighboring) under the user-level definition if they differ solely by the inclusion or exclusion of the complete interaction data from a single user $u$, such as all movie ratings, views, or preferences associated with that user (Pelikán et al., 2023, p. 3; Wang et al., 2021, p. 883). This add/remove-style adjacency captures the scenario where $D'$ can be obtained from $D$ by adding or removing all examples linked to one user, ensuring that the presence or absence of any individual's full profile minimally impacts the output model distribution (Pelikán et al., 2023, p. 3). Such a definition is particularly appropriate for recommendation systems, where users' complete interaction histories—encompassing viewing patterns, genre preferences, and temporal behaviors—represent highly sensitive information that could reveal personal tastes, lifestyles, or even demographics if compromised (Minto et al., 2021a, p. 1, 2021b; Na et al., 2025, p. 2).

This user-level privacy guarantee is inherently stronger than item-level privacy, which only considers datasets adjacent if they differ by a single user-item interaction (e.g., one rating or click) (Pelikán et al., 2023, p. 3). Item-level DP protects individual records but fails to shield holistic user profiles, allowing adversaries to reconstruct preferences by aggregating multiple inferences across sparse interactions. In contrast, user-level DP provides more meaningful protection in personalized scenarios like movie recommendations, where an adversary's ability to infer a user's overall profile (e.g., "horror enthusiast" or "family-oriented viewer") poses greater risks than isolated item leaks (Minto et al., 2021, p. 1). Empirical studies confirm that user-level local DP in federated recommenders achieves viable utility (e.g., HR@10 > 0.5 on MovieLens) while bounding inference risks (Minto et al., 2021a, p. 1, 2021b).

The DP-SGD mechanism operationalizes these guarantees through two key operations tailored to the user-level adjacency: per-sample gradient clipping and Gaussian noise addition (Bayrooti et al., 2023, p. 5; Ling et al., 2024, p. 4). During local training on a user's device, gradients for each interaction sample $\xi_j$ are computed as $g_{i,j}^t = \nabla F_i(w_i^t, \xi_j)$, then clipped to a maximum norm $C$: $\hat{g}_{i,j}^t = g_{i,j}^t / \max(1, \|g_{i,j}^t\| / C)$, bounding the $\ell_2$-sensitivity of individual contributions regardless of varying interaction counts per user (Ling et al., 2024, p. 4). Aggregated (user-level) noisy gradients are then uploaded, with noise $\sigma \propto C / \epsilon$ calibrated to the adjacency sensitivity $\Delta_2 \leq 2C$ (accounting for add/remove) (Bayrooti et al., 2023, p. 5). This ensures $(\epsilon, \delta)$-DP holds even for heterogeneous user datasets, as the clipping normalizes the influence of any single user's full data batch (Lin et al., 2025, p. 5).

To further bolster defenses against sophisticated inference attacks—such as membership inference (determining if a user's data was used), attribute inference (guessing demographics from preferences), or gradient inversion (reconstructing ratings from updates)—adaptive privacy mechanisms dynamically allocate budgets based on model vulnerabilities (Ju et al., 2025, p. 6; Zhang et al., 2023, p. 992). For instance, parameters with low attack resistance (e.g., user embeddings prone to inversion) receive larger noise factors (smaller $\epsilon$ shares), while robust components (e.g., global item biases) get minimal perturbation, optimizing the privacy-utility trade-off without uniform noise (S. Zhang et al., 2023, p. 992; X. Zhang et al., 2024, p. 2). Techniques like multi-agent reinforcement learning or Wasserstein-based heterogeneity estimation further refine per-round allocations, reducing degradation in sparse recommendation data (Chen et al., 2024; Zhu et al., 2025).

Moreover, the choice of adjacency relation demands careful consideration, as it profoundly affects noise calibration for target $\epsilon$. The add/remove relation (size-changing) suits FL's dynamic participation but complicates fixed-batch analyses (Ahmadi et al., 2025, p. 7). The replace-one relation maintains dataset size by swapping one user's data but often requires roughly double the noise, as it equates to delete-then-add (Annamalai et al., 2024, p. 2; Birrell et al., 2024, p. 3). Frameworks like FSRDP excel here, providing tighter bounds: under replace-one, FSRDP matches RDP noise levels, while for add/remove, it needs modest increases but guarantees both notions simultaneously—ideal for debatable real-world assumptions (Ahmadi et al., 2025, p. 7; Birrell et al., 2024, p. 3).

Specifically, in cross-device FL, a zero-out definition refines neighboring datasets for practicality: absent client updates are simulated by an all-zero vector, preserving fixed sizes without extra noise (semantically equivalent to add/remove) (Annamalai et al., 2024, p. 2; Chua et al., 2024, p. 5; Proceedings of the 61st Annual Meeting of the Association for Computational Linguistics (Volume 5: Industry Track), 2023, p. 651). This "ghost record" approach—replacing a user's contribution with $\perp \to 0$—simplifies accounting (e.g., zCDP or PLD) under participation limits like MaxP/MinS separations, preventing misbehaving clients from harming guarantees (McMahan et al., 2024, p. 2; Proceedings of the 61st Annual Meeting of the Association for Computational Linguistics (Volume 5: Industry Track), 2023, p. 651). Our implementation adopts this hybrid, enabling robust DP over multiple rounds while minimizing utility loss in resource-constrained mobile settings.

Evaluation Metrics

This section delineates the comprehensive evaluation framework employed to rigorously assess both the performance and privacy guarantees of our privacy-preserving federated learning system for mobile movie recommendation. In privacy-enhanced recommender systems, a balanced assessment is paramount, as differential privacy mechanisms like DP-SGD introduce controlled noise to protect user interaction histories—such as movie ratings and viewing patterns—while striving to maintain high recommendation utility amid heterogeneous mobile data distributions (Minto et al., 2021; Zhang et al., 2023, p. 992). Our framework adopts a multifaceted, dual-pronged approach, evaluating recommendation quality metrics that capture model accuracy, ranking efficacy, and predictive fidelity, alongside privacy metrics that quantify protection strength and empirical resilience against real-world threats (Andrew et al., 2023; Shiva & Isabel, 2025). This holistic methodology facilitates a precise characterization of the privacy-utility trade-off, enabling comparisons with baselines and insights into scalability under varying $\epsilon$ budgets and client participation rates, which is central to deploying effective systems in resource-constrained environments (Minto et al., 2021; Neumann et al., 2023).

Recommendation quality metrics, detailed in the following subsections, include ranking-oriented measures like Normalized Discounted Cumulative Gain (NDCG@10) and Hit Rate (Hit@10), which assess top-$k$ list relevance and position sensitivity; classification metrics such as Precision@10, Recall@10, and F1-score for sparse implicit feedback scenarios; and regression metrics including Mean Squared Error and Mean Absolute Error for explicit rating prediction (Peng et al., 2025; Shani & Gunawardana, 2010; Zhang et al., 2024, p. 8). These are computed on temporally split held-out test sets (e.g., 20% of interactions) to evaluate generalization to unseen user-movie pairs, ensuring temporal realism in mobile streaming contexts (Li et al., 2024, p. 11).

Privacy metrics encompass theoretical guarantees, such as the cumulative $(\epsilon, \delta)$-DP budget tracked via Rényi DP accountants or privacy loss curves across training rounds, alongside empirical audits like one-shot privacy estimation to verify real-world leakage without retraining (Andrew et al., 2023; Zhang et al., 2024, p. 3). We further probe vulnerability through adversarial simulations, measuring attack success rates for membership inference attacks, attribute inference attacks, and gradient inversion, benchmarking against non-private baselines to highlight defense efficacy (Yuan et al., 2023; Zhang et al., 2023, p. 994). By plotting utility degradation against tightening privacy budgets (e.g., $\epsilon \in \{0.1, 1, 10\}$), our analysis underscores optimal noise calibration for movie recommenders, where user-level DP bounds holistic profile inference risks without excessive utility loss (Minto et al., 2021a, p. 1, 2021b).

Recommendation Quality Metrics

To evaluate the recommendation quality of our federated learning system, we employ a suite of standard metrics commonly used in recommender systems research, each capturing different aspects of model performance. These metrics are computed on the held-out test set, which comprises 20% of the dataset temporally separated from the training data, ensuring an unbiased assessment of the model's generalization capabilities to unseen user-item interactions (Li et al., 2024, p. 11). Specifically, we utilize Hit Rate and Normalized Discounted Cumulative Gain to assess the model's ability to accurately retrieve relevant items and rank them appropriately within a personalized recommendation list (Na et al., 2025, p. 8). Furthermore, to provide a more holistic understanding of recommendation performance, we also analyze precision, recall, and F1-score, especially considering their utility in scenarios with sparse interaction data and varying user preferences. 

Normalized Discounted Cumulative Gain (NDCG@10)

Normalized Discounted Cumulative Gain (NDCG@10) evaluates the ranking quality of top-10 recommendation lists by rewarding relevant items higher in the list and penalizing poor positioning, using a logarithmic discount that reflects users' preference for top-ranked suggestions (Dacrema et al., 2022; Imran et al., 2022, p. 16; Shani & Gunawardana, 2010). Unlike Precision@10, which ignores order, or Hit Rate@10, which only checks presence, NDCG@10 captures both relevance and position, making it ideal for movie recommenders where users rarely scroll beyond the first few items (Misztal-Radecka & Indurkhya, 2021; Wang et al., 2021, p. 887; Zhang et al., 2024). In federated learning with implicit feedback, relevance $rel_i$ is binary (1 if the item is in the user's test interactions, 0 otherwise), computed on temporally held-out sets for realistic assessment (Jeunen et al., 2023; Li et al., 2024).

The metric proceeds in steps: Rank candidate movies by predicted scores to get the top-10 list. Compute Discounted Cumulative Gain (DCG@10) as $\text{DCG@10} = \sum_{i=1}^{10} \frac{rel_i}{\log_2(i+1)}$, where for binary relevance, this simplifies from the graded form $\frac{2^{rel_i}-1}{\log_2(i+1)}$ since $2^1-1=1$ and$2^0-1=0$ (Dacrema et al., 2022; Imran et al., 2022, p. 16; Wang et al., 2013). Compute Ideal DCG (IDCG@10) by rearranging the top-10 (or fewer if limited relevants) in perfect descending relevance order:

$$\text{IDCG@10} = \sum_{i=1}^{r} \frac{1}{\log_2(i+1)}$$

where $r \leq 10$ is the number of relevants (Dacrema et al., 2022, p. 7). Normalize per-user: $\text{NDCG@10}_u = \frac{\text{DCG@10}_u}{\text{IDCG@10}_u} \in$, then average: $\text{NDCG@10} = \frac{1}{|U|}\sum_u \text{NDCG@10}_u$ (Shani & Gunawardana, 2010; Tamm et al., 2021).

Its normalization ensures comparability across users with different interaction sparsity—crucial in non-IID FL data—and position sensitivity highlights utility drops from DP noise, which can shuffle rankings (Jeunen et al., 2023; Misztal-Radecka & Indurkhya, 2021; Wang et al., 2021). Limitations include reliance on binary grading (less nuanced than explicit ratings) and potential bias toward popular items in sparse settings, though it outperforms flat metrics in discriminating strong rankers (Jeunen et al., 2023; Tamm et al., 2021; Wang et al., 2013).

Hit Rate (Hit@10)

Hit Rate@10 (HR@10), also known as Hit@10, quantifies the proportion of users for whom at least one relevant item appears in the model's top-10 ranked recommendation list. This metric offers a straightforward, user-oriented assessment of recommendation success, emphasizing whether the system identifies any engaging content—crucial for movie recommenders where users often prioritize a single appealing suggestion over exhaustive coverage, especially in implicit feedback scenarios with sparse interactions (Elahi & Zirak, 2024, p. 4; Kaur & Shah, 2024, p. 4; Shani & Gunawardana, 2010). Unlike Recall@10, which accounts for the fraction of all relevant items retrieved, HR@10 saturates after the first hit, aligning closely with user satisfaction and engagement proxies like click-through rates in federated learning environments perturbed by differential privacy noise (Minto et al., 2021; Valcarce et al., 2020).

The evaluation process mirrors NDCG@10: For each user $u$ in the temporally held-out test set, compute predicted scores for all candidate movies and select the top-10 list $Top10_u$. Let $GT_u$ be the user's ground-truth relevant items (e.g., positive interactions post-split). A "hit" occurs if $Top10_u \cap GT_u \neq \emptyset$; otherwise, it is a miss. Assign a binary indicator $\mathbb{1}_{hit,u} = 1$ for hits and 0 for misses. Aggregate user-wise:

$$\text{HR@10} = \frac{1}{|U|} \sum_{u \in U} \mathbb{1}_{hit,u},$$

where $U$ is the test user set, yielding a value in $\epsilon \text{ and } k$ (Elahi & Zirak, 2024, p. 4; Jeunen et al., 2023; Minto et al., 2021). Metrics are averaged across multiple trials to mitigate stochasticity from client sampling and noise in federated settings (Valcarce et al., 2020).

HR@10 excels in simplicity and robustness to varying relevant item counts per user—ideal for non-IID federated data and MovieLens-like sparsity—while providing high discriminative power against baselines and sensitivity to privacy-induced utility degradation without position confounding (Minto et al., 2021; Shani & Gunawardana, 2010; Valcarce et al., 2018). It is widely adopted in privacy-aware recommenders for benchmarking on held-out sets, correlating well with real-world adoption (Ammad-ud-din et al., 2019; Kaur & Shah, 2024, p. 4).

Limitations include position insensitivity (treating rank-1 and rank-10 hits equally), failure to reward multiple hits, and vulnerability to popularity bias in long-tail distributions, potentially overestimating weak models that luck into one popular relevant item (Jeunen et al., 2023; Valcarce et al., 2018, 2020). These shortcomings necessitate pairing with ranking-aware metrics like NDCG@10 for comprehensive analysis, particularly under tightening $\epsilon$ budgets where noise flattens predictions (Minto et al., 2021).

Precision@10 and Recall@10

Precision@10 and Recall@10 are core set-based metrics borrowed from information retrieval to assess top-N recommendation lists in implicit feedback settings, quantifying the purity of recommendations (Precision@10: fraction of top-10 items that are relevant) and retrieval coverage (Recall@10: fraction of all user-relevant items appearing in top-10)—particularly valuable for sparse movie data under federated non-IID distributions (Anelli et al., 2021; Castells & Moffat, 2022; Tamm et al., 2021; Valcarce et al., 2020). Unlike position-aware NDCG@10 or binary Hit Rate@10, they treat top-10 slots equally, simplifying analysis but ignoring rank nuances in user interfaces (Valcarce et al., 2018, 2020).

The process aligns with NDCG@10 and HR@10 protocols: For each user $u$ in the temporally held-out test set, score all candidate movies, rank to form $Top10_u$, with $GT_u$ as binary ground-truth relevants. Let $hits_u = |Top10_u \cap GT_u|$. Per-user metrics are $\text{Precision@10}_u = \frac{hits_u}{10}$ and $\text{Recall@10}_u = \frac{hits_u}{|GT_u|}$ if $|GT_u| > 0$, else 0 (Chakma et al., 2025; Rattaphun et al., 2021; Tamm et al., 2021). Aggregate via macro-averaging: $\text{Precision@10} = \frac{1}{|U|} \sum_{u \in U} \text{Precision@10}_u$$\{u \in U\}$ $\text{Precision@10}_u$, similarly for Recall@10, where $U$ is the test users (Flanagan et al., 2021; Valcarce et al., 2020). Average over trials accounts for FL/DP stochasticity like client sampling and noise (Anelli et al., 2021; Castells & Moffat, 2022).

Precision@10 offers high robustness to sparsity and popularity biases, resisting score flattening from DP noise, while Recall@10 sensitively tracks coverage loss for long-tail items; their harmonic mean $F1@10 = 2 \cdot P \cdot \frac{R} {P + R}$ balances both for holistic utility (Chakma et al., 2025; Tamm et al., 2021; Valcarce et al., 2020). Limitations: position insensitivity (e.g., intra-top-10 swaps unchanged), vulnerability to lucky popular hits, and sensitivity to varying $|GT_u|$—demanding complementarity with NDCG@10/HR@10 for full evaluation under privacy budgets (Castells & Moffat, 2022; Valcarce et al., 2018, 2020). In experiments, we report both alongside F1@10 (Gong et al., 2023; Hasan et al., 2024).

Regression Metrics: MSE, MAE, and Accuracy

In addition to ranking-based metrics, we also evaluate the model's ability to predict exact rating values using regression metrics. Mean Squared Error (MSE) measures the average squared difference between predicted and actual ratings:

$$MSE = \frac{1}{n} \sum_{i=1}^{n} (y_i - \hat{y}_i)^2$$

where $y_i$ is the actual binarized rating (1.0 or 0.0) and $\hat{y}_i$ is the predicted rating value. Mean Absolute Error (MAE) provides a more interpretable measure of prediction error: 

$$MAE = \frac{1}{n} \sum_{i=1}^{n} |y_i - \hat{y}_i|$$

While our model is primarily optimized for ranking tasks, these regression metrics provide insights into the calibration of the model's predictions and its ability to distinguish between positive and negative interactions. Additionally, we compute binary classification accuracy by thresholding the predicted ratings at 0.5: $Accuracy = \frac{1}{n} \sum_{i=1}^{n} \mathbb{1}[(y_i > 0.5) = (\hat{y}_i > 0.5)]$, which measures the fraction of test samples for which the model correctly predicts the binary relevance label. These metrics are particularly useful for understanding how differential privacy mechanisms affect the model's predictive accuracy, as noise addition can degrade both ranking quality and prediction accuracy, and these metrics help quantify the magnitude of such degradation (Sharma, 2023).

Privacy Metrics

The privacy guarantees provided by our DP-SGD implementation are quantified through the privacy budget parameter epsilon ($\epsilon$), which serves as the primary privacy metric in our evaluation framework. The epsilon value directly measures the privacy loss incurred during training, with lower values indicating stronger privacy protection. In our experiments, we evaluate the system across multiple privacy budget levels: $\epsilon \in \{∞, 8, 4, 2, 1\}$, where $\epsilon = ∞$ corresponds to no differential privacy protection (baseline case), and progressively lower epsilon values represent increasingly stringent privacy guarantees. The delta parameter ($\delta$) is fixed at $10^{-5}$ across all experiments, representing the probability of privacy failure, which is set to be negligibly small compared to the inverse dataset size to ensure meaningful privacy protection. The actual privacy budget consumed during training is tracked using the RDP accountant, which computes the cumulative privacy loss across all training rounds and batches, ensuring that the total privacy cost does not exceed the target epsilon value. This privacy accounting is essential for providing formal privacy guarantees and understanding the relationship between privacy budget allocation and model utility (Abadi et al., 2016; Schwethelm et al., 2024, p. 4).

Evaluation Methodology

Our evaluation methodology follows a rigorous protocol designed to ensure fair and reproducible comparisons across different privacy configurations and experimental conditions. All metrics are computed on the same held-out test set, which is temporally separated from the training data to prevent data leakage and ensure realistic evaluation scenarios. For each experiment configuration (specified by epsilon, alpha, and other hyperparameters), we run multiple trials with different random seeds (typically 3 seeds: 42, 123, and 456) to account for stochastic variability in the training process, model initialization, and data partitioning. The final metrics reported for each configuration are the mean values across all trials, with standard deviations provided to indicate the variance in results. This multi-trial approach is particularly important in federated learning settings where non-deterministic factors such as client selection order, data partitioning randomness, and noise generation can introduce variability in model performance (Kuo et al., 2022; Xu et al., 2023, p. 9).

Evaluation happens at the end of each federated learning round. This lets us track how the model's performance changes over time and see how it converges under different privacy rules.

Final metrics are calculated after all training rounds finish, giving a full picture of the trained model's performance.

For recommendation metrics like NDCG@10, Hit@10, Precision@10, and Recall@10, the process follows these steps:





Group test interactions by user.



For each user, score all items in the catalog using the trained model.



Rank items by their predicted scores.



Take the top 10 recommendations.



Calculate metrics based on how relevant those top 10 items are.



Average the metrics across all users.

This per-user approach makes sure the metrics show real performance for individual users, which matters most in recommendation systems. This methodology provides a robust framework for assessing the trade-offs between accuracy and privacy in federated learning for mobile movie recommendation systems.

Normalized Discounted Cumulative Gain (NDCG@10) 

NDCG@10 is a ranking metric that checks how good the top 10 recommendations are. It looks at both how relevant the items are and their position in the list—items higher up count more, thanks to a log-based discount.

This metric works well for recommendations where order matters, as it punishes relevant items that show up too low.

The score is calculated for each user by ranking all items by predicted scores, then computing the Discounted Cumulative Gain (DCG) for the top 10:

$$\text{DCG@10} = \sum_{i=1}^{10} \frac{rel_i}{\log_2(i+1)} $$

Here, $rel_i$is the relevance at position $i$ (1.0 if rating ≥ 4.0, 0.0 otherwise).

DCG is normalized by the Ideal DCG (IDCG), which is the DCG for a perfect order with all relevant items at the top:

$$\text{NDCG@10} = \frac{\text{DCG@10}}{\text{IDCG@10}} $$

The final NDCG@10 is the average across all test users. It ranges from 0.0 (worst) to 1.0 (perfect). This metric evaluates both whether relevant items are included and how well they are ranked (Frihat & Fuhr, 2024, p. 10; Wang et al., 2013; Zhang et al., 2016, p. 131). 

Hit Rate (Hit@10) 

Hit@10, also referred to as Recall@10 in some contexts, measures the fraction of users for whom at least one relevant item appears in the top-10 recommendations. This metric is particularly useful for evaluating recommendation systems in scenarios where user satisfaction depends on finding at least one relevant item, such as in movie recommendation where users typically seek engaging content rather than an exhaustive list of all relevant movies. For each user, Hit@10 is computed as a binary indicator: 1 if at least one relevant item appears in the top-10 recommendations, and 0 otherwise. The final Hit@10 metric is the average of these binary indicators across all users: $Hit@10 = \frac{1}{|U|} \sum_{u \in U} \mathbb{1}[\exists i \in Top10(u) : item_i \text{ is relevant}]$, where $U$ is the set of users and $Top10(u)$ represents the top-10 items recommended to user $u$. This metric provides a user-centric perspective on recommendation quality, focusing on whether the system successfully identifies relevant content for individual users, and is widely used in recommendation system evaluation as it correlates well with user satisfaction and engagement metrics (Kaur & Shah, 2024, p. 4; Luo et al., 2024, p. 15; Shani & Gunawardana, 2010).

Precision@10 and Recall@10

Precision@10 measures the fraction of recommended items in the top-10 list that are actually relevant, serving as a key indicator of recommendation accuracy by quantifying the proportion of true positives among all top-10 suggestions and penalizing false positives. Formally, for each user $u$, it is computed as

$$Precision@10(u) = \frac{|Top10_u \cap Rel_u|}{10},$$

where $Top10_u$ denotes the set of the top-10 items ranked by predicted scores for user $u$, and $Rel_u$ is the set of relevant items for $u$ from the held-out test set (typically defined via binary relevance, such as ratings $\geq 4$ or implicit positive interactions in movie recommendation contexts)(Jadon & Patil, 2023, p. 15; Lee et al., 2022, p. 2; Tamm et al., 2021, p. 709).

Recall@10 complements Precision@10 by assessing the fraction of all relevant items successfully retrieved within the top-10 recommendations, highlighting the model's coverage and ability to minimize false negatives:

$$Recall@10(u) = \frac{|Top10_u \cap Rel_u|}{|Rel_u|}.$$

Both metrics are evaluated per-user on the temporally split test set and then macro-averaged across all test users to yield aggregate scores:

$Precision@10 = \frac{1}{|U|} \sum_{u \in U} Precision@10(u), \quad Recall@10 = \frac{1}{|U|} \sum_{u \in U} Recall@10(u),$

where $U$ is the set of test users(Jadon & Patil, 2023, p. 15; Kuanr & Mohapatra, 2021, p. 402; Lee et al., 2022, p. 2). High Precision@10 signifies precise targeting with minimal irrelevant suggestions, ideal for user satisfaction in sparse recommendation scenarios like mobile movie apps, while high Recall@10 ensures broad discovery of user-preferred content, crucial for engagement(Kuanr & Mohapatra, 2021, p. 402; Tamm et al., 2021, p. 709).

These metrics provide complementary, user-centric views on performance: optimizing one often trades off against the other, motivating their joint reporting (sometimes via F1-score as their harmonic mean). Notably, unlike position-aware metrics such as NDCG@10—which logarithmically discounts lower positions to prioritize top-ranked relevance—Precision@10 and Recall@10 are position-insensitive within the top-10, treating all slots equally and focusing solely on inclusion rather than order (Lee et al., 2022, p. 2; Tamm et al., 2021, p. 709; Valcarce et al., 2020). This makes them robust to minor ranking perturbations but less sensitive to exact positioning, suiting evaluations where top-K coverage trumps fine-grained ranking. In our privacy-preserving federated learning experiments, both are computed post-training to dissect the utility degradation from DP noise and federated heterogeneity across multiple $\epsilon$ levels, offering a comprehensive angle alongside ranking and regression metrics(Valcarce et al., 2020).

$$MSE = \frac{1}{n} \sum_{i=1}^{n} (y_i - \hat{y}_i)^2,$$

where $y_i$ is the actual binarized rating (1.0 or 0.0) and $ \hat{y}_i $ is the predicted rating value.

Mean Absolute Error (MAE) gives a clearer measure of prediction error:

Although our model is mainly trained for ranking tasks, these regression metrics help us see how well the predictions are calibrated and how clearly the model separates positive and negative interactions.

The privacy guarantees provided by our DP-SGD implementation are quantified through the privacy budget parameter $\epsilon$, which serves as the primary privacy metric, complemented by the fixed failure probability $\delta = 10^{-5}$ across all experiments. This $\delta$ represents a negligibly small probability of privacy failure, typically set much smaller than the inverse of the dataset size (e.g., $\frac{1}{|D|}$) to ensure meaningful protection against adversarial inference(Dwork & Feldman, 2018; Lefkovitz, 2024).

The actual privacy budget consumed during training is precisely tracked using the Rényi Differential Privacy accountant, a state-of-the-art privacy accounting mechanism that computes the cumulative privacy loss across all federated training rounds, local epochs, and mini-batches(Dörmann et al., 2021, p. 2; Feng et al., 2024, p. 3; Wang et al., 2021, p. 884). In DP-SGD, this involves two key modifications per iteration: clipping per-example gradients to a norm bound $\mathcal{C}$ (e.g., $\mathcal{C}=1.0$) to limit sensitivity, and adding zero-mean Gaussian noise $\mathcal{N}(0, \sigma^2 \mathcal{C}^2 I)$ to the averaged clipped gradients, where the noise multiplier $\sigma$ (e.g., $\sigma=0.5$) is tuned to achieve target RDP parameters(Dörmann et al., 2021, p. 2; Saifullah et al., 2023, p. 8; Xie et al., 2023, p. 1517). The RDP accountant exploits the additive composition of RDP for order $\alpha > 1$: if each step satisfies $(\alpha, \rho)$-RDP, the total over $T$ steps is $(\alpha, T\rho)$-RDP, enabling tight tracking via the Rényi divergence $D_\alpha(P \| Q) = \frac{1}{\alpha-1} \log \mathbb{E}_{x \sim Q} \left[ \left( \frac{P(x)}{Q(x)} \right)^\alpha \right] $(Feng et al., 2024, p. 3; Rinberg et al., 2025, p. 4; Yang, 2025, p. 6). The accumulated RDP is then converted to $(\epsilon, \delta)$-DP bounds using $\epsilon(\delta) = \min_{\alpha \in A} \frac{\mathrm{RDP}_\alpha + \log(1/\delta)}{\alpha - 1}$, where $A$ is a discrete set of $\alpha$ values, yielding tighter guarantees than earlier moments accountants(Da et al., 2022, p. 3; Kairouz et al., 2021; Wang et al., 2021, p. 884).

This rigorous privacy accounting via the Rényi Differential Privacy accountant is essential for delivering formal $(\epsilon, \delta)$-DP guarantees in our peer-to-peer federated learning setting, where client data heterogeneity induces divergent local gradient norms and multiple communication rounds lead to extensive compositions of DP mechanisms, thereby substantially amplifying cumulative privacy loss relative to centralized training(Feng et al., 2024, p. 3; Wei et al., 2024, p. 5; Xin-wei et al., 2022, p. 3). Specifically, heterogeneity exacerbates sensitivity through non-IID distributions that cause client drift during local epochs, while repeated peer-to-peer exchanges compound RDP orders additively—${\rm RDP}_{\alpha}^{\rm total} = \sum_t {\rm RDP}_{\alpha}(q_t, \sigma)$ across $T$ steps—necessitating tight tracking to avoid overestimation via naive composition(Rinberg et al., 2025, p. 4; Yang, 2025, p. 6; Yuan et al., 2024, p. 4). By dynamically monitoring privacy expenditure after each iteration (factoring in per-example clipping to norm $\mathcal{C}$, Gaussian noise $\mathcal{N}(0, \sigma^2 \mathcal{C}^2)$, Poisson subsampling rate $q$, and RDP-to-DP conversion

$$\epsilon(\delta) = \min_{\alpha \in A} \frac{{\rm RDP}_{\alpha} + \log(\frac{1}{\delta})}{\alpha - 1}$$

 we ensure the total budget stays strictly below target levels $\epsilon \in \{\infty, 8, 4, 2, 1\}$ with fixed $\delta=10^{-5}$ throughout training(Toubeau et al., 2022, p. 804; Wang et al., 2021, p. 884; Xie et al., 2023, p. 1517). This precise calibration not only prevents privacy overspending but also enables detailed dissection of privacy-utility trade-offs—such as noise-induced degradation in ranking metrics (e.g., $\Delta{\rm NDCG}@10 \approx 10\%$ at $\epsilon=1$ due to amplified variance)—and reproducible comparisons against baselines like centralized non-private models or alternative federated schemes(Andrew et al., 2023; Imran et al., 2022, p. 19; Ribero et al., 2022, p. 108759). The Rényi accountant, by tracking the cumulant generating function of the privacy loss random variable, enables a more precise estimation of cumulative privacy loss in complex scenarios involving multiple differentially private operations over time, such as those encountered in federated learning (Feng et al., 2024, p. 3). Compared to DP-FedAvg, our approach utilizing the Rényi Differential Privacy mechanism offers tighter privacy bounds, which are better suited for the distributed training paradigm of federated learning (Wang et al., 2025, p. 6).

Statistical Significance and Reporting

To ensure robust statistical analysis and reproducibility in our federated learning experiments with differential privacy, we conduct 5 independent runs for each configuration using distinct random seeds. This practice accounts for variability introduced by stochastic elements such as mini-batch sampling in DP-SGD, client selection in federated rounds, data shuffling across non-IID distributions, and the inherent randomness of Gaussian noise mechanisms calibrated via the RDP accountant (Morafah et al., 2023). We systematically report mean values ± standard deviations for all key metrics (e.g., NDCG@10, Recall@10, MRR, and binary accuracy), facilitating quantitative assessment of result consistency and reliability. Elevated standard deviations signal potential instability, sensitivity to initialization (e.g., weight scaling or embedding dropout), or amplified degradation under tight privacy budgets ($\epsilon < 2$), which is particularly pronounced in heterogeneous recommendation data where label skew exacerbates variance (Chai et al., 2020, 2024, p. 5010).

When contrasting configurations—such as federated vs. centralized training, varying $\epsilon \in \{\infty, 8, 4, 2, 1\}$, or baseline models—we evaluate both absolute performance differences (e.g., $\Delta$NDCG = 0.05 drop at $\epsilon=1$) and relative degradations (e.g., 12% utility loss vs. non-private baseline), complemented by statistical tests like paired t-tests (p < 0.05 threshold) or Wilcoxon signed-rank tests for non-parametric validation of significance. This multifaceted analysis elucidates the nuanced impacts of privacy mechanisms on utility, aligning with holistic FL evaluation frameworks that emphasize multi-dimensional benchmarking across effectiveness, robustness, and privacy (Chai et al., 2020, 2024, p. 5010).

The experimental results are conveyed through an array of intuitive visualizations: line plots tracking metric evolution (e.g., NDCG@10 progression) across communication rounds, revealing convergence speed, oscillation damping from local epochs, and noise-induced plateaus at low $\epsilon$; grouped bar charts juxtaposing final metrics by privacy budget, data partition (IID/non-IID), and baselines, with error bars denoting standard error of the mean ($\sigma / \sqrt{5}$); and scatter plots mapping the privacy-utility trade-off (e.g., $\epsilon$ on x-axis vs. averaged NDCG/Recall on y-axis), often augmented with polynomial trend lines or Pareto frontiers to quantify degradation slopes (e.g., ~5% utility loss per halving of $\epsilon$) (Andrew et al., 2023; Saifullah et al., 2024, p. 19). These graphical aids, drawn from established practices in DP-FL benchmarking, illuminate intricate dependencies among privacy parameters, training hyperparameters (e.g., clipping norm $\mathcal{C}=1.0$, noise multiplier $\sigma=0.5$), and performance outcomes, directly informing the in-depth interpretations in the Results and Discussion section (Arasteh et al., 2023; Mohammadi et al., 2025).

Experimental Setup

This section details the specific experimental configuration, simulation environment, hyperparameter settings, and baseline comparisons employed to systematically evaluate the accuracy–privacy trade-offs in federated learning for mobile movie recommendation systems (Ammad-ud-din et al., 2019; Neumann et al., 2023; Qin et al., 2023, p. 11). The experimental design adheres to rigorous, reproducible methodologies established in federated recommendation research, enabling evaluation across diverse privacy budgets ($\epsilon=1-15$, $\delta=10^{-5}$) and data heterogeneity levels (Dirichlet concentration $\alpha=0.1-1.0$), while capturing real-world heterogeneities like user preference skews and sparse interactions on benchmarks including MovieLens-1M, HetRec-MovieLens, and Amazon-Video (Morafah et al., 2023, p. 1708; Na et al., 2025, p. 5; Wan et al., 2023, p. 5; Wang et al., 2021, p. 880). This facilitates comprehensive analysis of key research questions, such as federated vs. centralized degradation ($\Delta$NDCG $\approx3-7\%$), DP-induced utility loss (5-10% NDCG drop at $\epsilon=2$), and entropy-adaptive advantages in convergence and metrics like Recall@10/HR@10 (Chai et al., 2024, p. 5010; Minto et al., 2021).

The simulation environment implements a scalable client-server federated averaging architecture using Python 3.8+, PyTorch 1.12+ for local training, and FastAPI for RESTful communication, simulating realistic network patterns and hybrid deployments with mobile emulators while abstracting physical constraints (McMahan et al., 2017; Qin et al., 2023, p. 11). Each round follows a synchronous protocol: clients download global models (JSON/base64 serialized), perform local updates, upload parameters, and the server aggregates via weighted FedAvg (Maheri et al., 2025, p. 9).

Hyperparameters are tuned via grid search on validation splits with 5 independent runs per configuration (distinct seeds), aligning with best practices: embedding dimension 16, learning rate $\eta=10^{-3}-0.01$, batch size $B=32-128$, local epochs $E=1-5$, clipping norm $\mathcal{C}=1.0$, and noise multiplier $\sigma=0.5-1.0$ dynamically computed using RDP accountants to match target $(\epsilon,\delta)$-DP guarantees (Cai et al., 2024, p. 7; Dwork & Roth, 2014; Morafah et al., 2023, p. 1711). Data splits employ leave-one-out or temporal holds, with non-IID partitioning via Dirichlet to model clustering (Na et al., 2025, p. 5).

Baselines are selected for native peer-to-peer support, DP integration, and public code: non-DP FedAvg, centralized (upper bound), DP-FedAvg/DP-DSGT, Scaffold, FedGNN, and PrivRec/DP-PrivRec (Maheri et al., 2025, p. 9; Wang et al., 2021, p. 878; Wu et al., 2021). Evaluations quantify trade-offs via NDCG@10/Recall@10/HR@10, with error bars ($\sigma/\sqrt{5}$), scatter plots ($\epsilon$ vs. metrics with trend lines), paired t-tests ($p<0.05$), and Wilcoxon tests for significance (Ammad-ud-din et al., 2019, p. 10; Andrew et al., 2023; Saifullah et al., 2024, p. 19).

Simulation Environment

The experimental framework operates within a scalable client–server federated learning architecture implemented using Python 3.8+ with PyTorch 1.12+ for efficient model training and FastAPI for asynchronous RESTful server communication, enabling high-throughput simulations of large-scale deployments (Burlachenko et al., 2022; McMahan et al., 2017; Pinyoanuntapong et al., 2022, p. 109404; Qin et al., 2023, p. 11). This stack leverages PyTorch's dynamic computation graphs and automatic differentiation for local optimization on heterogeneous client data, while FastAPI's support for WebSockets and HTTP/2 protocols facilitates low-latency, concurrent handling of multiple client requests, crucial for mimicking real-world edge computing constraints in recommendation systems (Dimitriadis et al., 2022; Pinyoanuntapong et al., 2022, p. 109404).

The central server component, hosted on a high-performance local machine (e.g., equipped with multi-core CPUs and GPUs), exposes a suite of RESTful API endpoints to orchestrate the federated process: /init for global model initialization and hyperparameter broadcasting; /register for client authentication and dynamic participation tracking; /upload for secure reception of clipped and noised parameter updates; and /download for distributing the latest aggregated global model (Guberović et al., 2022, p. 170; Han et al., 2022, p. 5). These endpoints incorporate rate limiting, asynchronous processing via ASGI, and JSON schema validation to ensure robustness against malformed requests and support scalability to hundreds or thousands of simulated clients without bottlenecks (Dimitriadis et al., 2022; Sani et al., 2023, p. 1).

Clients are emulated as lightweight, independent Python processes (or Docker containers for isolation), communicating synchronously with the server over HTTP requests using libraries like httpx or aiohttp for efficient, non-blocking I/O. This design allows scalable experimentation with variable numbers of participants (e.g., 10–1000 clients), incorporating configurable network delays, bandwidth limitations, and dropout rates to replicate realistic mobile heterogeneities such as intermittent connectivity and straggler effects (Bouraqqadi et al., 2021, p. 1; Cox et al., 2024, p. 1; Dimitriadis et al., 2022). For instance, client-side logic includes local data loading from non-IID partitions (via Dirichlet sampling), gradient computation with clipping ($\mathcal{C}=1.0$) and Gaussian noise addition for $(\epsilon, \delta)$-DP, and model serialization before upload (Morafah et al., 2023, p. 1711; Wang et al., 2021, p. 880).

This simulation environment effectively abstracts the complexities of physical mobile device deployment—such as battery drain, sensor noise, and OS-specific restrictions—while faithfully preserving core federated learning dynamics: strict decentralized data storage on clients, iterative local training epochs ($E=1-5$), secure parameter transmission (often compressed via quantization), and server-side weighted aggregation via FedAvg (Burlachenko et al., 2022; McMahan et al., 2017). Global model parameters are serialized in a portable JSON format using base64 encoding for cross-platform compatibility, supporting hybrid experiments where simulated Python clients interoperate with real Android emulators or Flutter/Dart mobile apps (Han et al., 2022, p. 5). Such flexibility validates performance under mixed workloads, aligning with benchmarks in privacy-preserving recsys like MovieLens, where convergence is tracked over 100+ rounds amid user skew and sparsity (Minto et al., 2021; Wang et al., 2021, p. 878).

The server implements the standard Federated Averaging algorithm (McMahan et al., 2017), where it receives clipped and noised local model updates from participating clients, computes a weighted average based on the number of samples per client, and broadcasts the updated global model back for the next round. This aggregation mechanism ensures convergence under heterogeneous data distributions while preserving privacy through local computation and secure parameter transmission (Burlachenko et al., 2022; Maheri et al., 2025, p. 9).

Global model parameters, consisting of user and item embedding matrices in the matrix factorization architecture, are serialized into a portable JSON format using base64 encoding. This approach transforms binary tensor data (e.g., float32 arrays) into ASCII-safe strings, preventing transmission issues over text-based RESTful APIs like those exposed by FastAPI, while maintaining human-readability for debugging and compatibility across programming languages without requiring specialized libraries or compilers (Han et al., 2022, p. 5; Zandberg et al., 2024). JSON's structured key-value pairs facilitate straightforward parsing on diverse clients, with base64 ensuring robustness against encoding mismatches in HTTP payloads, a common challenge in cross-platform federated setups (Viotti & Kinderkhedia, 2022).

This serialization strategy is particularly advantageous for heterogeneous client ecosystems, supporting seamless interoperability with mobile implementations in languages such as Dart (for Flutter apps) or Java/Kotlin (for native Android), alongside Python-based simulators (Beutel et al., 2020). It aligns with best practices in scalable FL frameworks like Flower and FL_PyTorch, which emphasize framework-agnostic communication for real-world deployments (Beutel et al., 2020; Burlachenko et al., 2022).

The design enables sophisticated hybrid experiments, where a large majority of clients are emulated as lightweight Python processes for high-throughput scalability (e.g., simulating thousands of participants), while a subset integrates real Android devices or emulators to capture authentic mobile constraints such as variable network latency, bandwidth throttling, battery drain, and CPU/GPU limitations (Dimitriadis et al., 2022). For instance, physical devices can be connected via Wi-Fi to the FastAPI server, validating end-to-end performance under straggler effects and dropout scenarios that purely software-based simulations might overlook (Cox et al., 2024, p. 1). This hybrid paradigm enhances experimental fidelity and reproducibility: simulations provide controlled, deterministic scaling for hyperparameter sweeps and statistical significance testing (e.g., 5 runs with error bars), while real-device integration offers grounded insights into deployment viability, mirroring challenges in privacy-preserving recommendation systems on edge devices (Lai et al., 2021). The matrix containing user preferences is factorized into a user matrix and an item matrix, from which a k-dimensional user-specific embedding is generated for each user (Chopra & Raja, 2024, p. 99). 

Each federated learning round adheres to the canonical synchronous Federated Averaging protocol (McMahan et al., 2017; McMahan, Moore, Ramage, & Arcas, 2016; McMahan, Moore, Ramage, Hampson, et al., 2016), comprising four core phases orchestrated via dedicated FastAPI endpoints to ensure efficient, scalable coordination among simulated clients. In the initial phase, participating clients—authenticated and tracked dynamically via the /register endpoint—retrieve the current global model parameters $\theta^t$ from the server using the /download endpoint. These parameters, encompassing both user and item embedding matrices for the matrix factorization architecture, are serialized into a portable JSON format with base64 encoding to guarantee cross-platform compatibility and robustness over HTTP transmission (Han et al., 2022, p. 5; Zandberg et al., 2024). Clients initialize their local models with $\theta^t$ and execute $E=1$ local training epochs on non-IID private data partitions (generated via Dirichlet distribution with $\alpha=0.5$), employing stochastic gradient descent with learning rate $\eta=0.01$ and batch size $B=32$ to minimize the reconstruction loss for user-item interactions (Ammad-ud-din et al., 2022; Burlachenko et al., 2022).

Following local optimization, clients apply gradient clipping at norm $\mathcal{C}=1.0$ and add Gaussian noise calibrated via an RDP accountant to enforce $(\epsilon, \delta=10^{-5})$-differential privacy (with $\sigma$ dynamically adjusted per privacy budget), before serializing and uploading the perturbed local updates $\Delta\theta_k^t$ via the /upload endpoint (Dwork & Roth, 2014; Wang et al., 2021, p. 880). Upon receiving updates from all active clients (accounting for configurable dropout and straggler effects), the server computes the weighted aggregation $\theta^{t+1} = \sum_{k=1}^K \frac{n_k}{N} (\theta^t + \Delta\theta_k^t)$, where $n_k$ denotes the number of local samples for client $k$ and $N = \sum n_k$ is the total across $K$ clients, embodying the FedAvg mechanism to mitigate non-IID heterogeneity while converging toward a privacy-preserving global optimum (Burlachenko et al., 2022; McMahan et al., 2016; Wang et al., 2021). The updated global model is then broadcast for the next round, with periodic evaluation on a held-out test set (e.g., MovieLens) after each of $T=10$ rounds to monitor key recommendation metrics such as Hit Rate and Normalized Discounted Cumulative Gain, thereby tracking convergence dynamics, utility degradation under privacy constraints, and proximity to centralized baselines (Minto et al., 2021; Wang et al., 2021, p. 878). The hyper-parameters for the model, including K, $\alpha$, $\lambda_1$, and $\lambda_2$, along with Adam's learning rate parameters ($\beta_1, \beta_2, \epsilon, \text{ and } \gamma$), and the federated learning threshold $\Theta$, are meticulously tuned using Bayesian Optimization to achieve optimal performance (Flanagan et al., 2021, p. 333). 

Hyperparameter Configuration

The experimental configuration employs a fixed set of hyperparameters across all federated learning experiments. These include an embedding dimension of $K=16$ for the matrix factorization model (covering both user and item embeddings), along with $\alpha=0.5$, $\eta=0.01$, $B=32$, $E=1$, and $C=1.0$, tuned via Bayesian optimization to optimize performance in federated recommender systems (Flanagan et al., 2021, p. 333). This$K=16$yields a compact architecture suitable for mobile deployment while providing adequate representational capacity for user-item interactions (Beutel et al., 2020; Burlachenko et al., 2022; Chopra & Raja, 2024, p. 99). The dimensionality balances expressiveness against computational demands, enabling efficient local inference on resource-constrained devices and reduced communication overhead in parameter transmission.

Training hyperparameters are fixed across all federated learning experiments as follows: learning rate $\eta = 0.01$, batch size $B = 32$, and local epochs $E = 1$ per federated round, selected via Bayesian optimization to optimize performance in federated recommender systems (Flanagan et al., 2021, p. 333). This configuration, with $K=16$ embedding dimension, $\alpha=0.5$, and $C=1.0$ gradient clipping, yields a compact architecture suitable for mobile deployment while balancing expressiveness, computational demands, and communication overhead (Beutel et al., 2020; Burlachenko et al., 2022; Chopra & Raja, 2024, p. 99). The choice of a single local epoch ($E=1$) adheres to the canonical synchronous Federated Averaging protocol (McMahan et al., 2017; McMahan, Moore, Ramage, & Arcas, 2016; McMahan, Moore, Ramage, Hampson, et al., 2016), limiting local computation to mitigate client drift—the divergence of local models from the global optimum due to non-IID data distributions—while relying on multiple global aggregation rounds for convergence (Karimireddy et al., 2019; Seo et al., 2025). In heterogeneous recommendation scenarios, excessive local epochs exacerbate drift, slowing convergence and degrading performance even with full-batch gradients, whereas $E=1$ maintains alignment with the global model under resource constraints typical of edge devices (Karimireddy et al., 2019; Seo et al., 2025). The batch size of $B=32$ balances high-quality gradient estimation with computational efficiency and low memory usage on mobile clients, introducing stochastic noise that acts as regularization to escape sharp minima, enhance generalization, and counter statistical heterogeneity (Moskvoretskii et al., 2024, p. 8; Perrone et al., 2019; Tyagi, 2025, p. 5). Larger batches improve gradient precision but risk overfitting and poor generalization in federated settings, whereas smaller ones like 32 promote robustness without excessive variance (Tyagi, 2025, p. 5). The learning rate $\eta=0.01$ ensures stable, oscillation-free convergence, empirically validated through preliminary experiments on the MovieLens dataset and consistent with successful federated collaborative filtering implementations (Ammad-ud-din et al., 2019; Minto et al., 2021; Moskvoretskii et al., 2024, p. 8).

For the differential privacy experiments, the gradient clipping norm is fixed at $C = 1.0$ across all privacy budget levels $\epsilon \in \{\infty, 8, 4, 2, 1\}$, ensuring a consistent $\ell_2$-sensitivity bound on per-sample (or per-client) gradients regardless of the target $\epsilon$ value, which simplifies privacy accounting and promotes training stability in federated settings (Andrew et al., 2019; Pelikán et al., 2023, p. 3; Weng et al., 2024, p. 22581). This fixed-norm approach, commonly adopted in DP-federated learning, contrasts with adaptive clipping methods that estimate quantiles of the gradient norm distribution online but requires no hyperparameter tuning beyond empirical validation on baselines like MovieLens (Andrew et al., 2019; Bu et al., 2022; Wei et al., 2021, p. 805). The $\delta$ parameter for $(\epsilon, \delta)$-DP is set to $\delta = 10^{-5}$, satisfying the standard heuristic $\delta \ll 1/n$ (where $n$ is the total training samples across clients) to yield negligible failure probability while enabling practical utility, as validated in diverse DP-SGD applications from speech recognition to NLP (Ahmadi et al., 2025, p. 6; Arasteh et al., 2024, p. 21; Jiménez-López et al., 2023, p. 5; Wang et al., 2021, p. 315).

The noise multiplier $\sigma$ is dynamically computed for each target $\epsilon$ via the Rényi Differential Privacy accountant, which tightly composes privacy loss over $T=10$ federated rounds by accounting for the Gaussian mechanism's RDP order $\alpha >1$, subsampling amplification from batch size $B=32$ (sampling rate $q \approx B/n_k$ per client), average samples per client $n_k$, full client participation, and clipping $C=1.0$, ensuring the realized $(\epsilon, \delta)$ closely matches the target without over- or under-spending the budget (Ahmadi et al., 2025, p. 3; Kairouz et al., 2021; Pelikán et al., 2023, p. 3; Wei et al., 2024, p. 5). This RDP-based calibration outperforms older moments accountants for Gaussian noise in iterative DP-SGD, especially under heterogeneous FL workloads (Malekmohammadi et al., 2024, p. 3; Yin et al., 2025, p. 10).

Data partitioning employs a Dirichlet distribution $\mathrm{Dir}(\alpha)$ over user-item interactions to generate controlled non-IID client splits, a standard benchmark for FL heterogeneity that flexibly simulates real-world skew via a single hyperparameter (Kais et al., 2023, p. 6; Milasheuski et al., 2024, p. 1020; Ohib et al., 2024, p. 7). Lower $\alpha$ (e.g., $0.1$) produces highly heterogeneous distributions—clients receive skewed proportions, specializing in niche user groups or item categories (e.g., specific genres in MovieLens)—exacerbating client drift (Barbieri et al., 2024, p. 4; Park et al., 2025, p. 28); higher $\alpha$ (e.g., $1.0$) yields near-uniform, IID-like splits with balanced coverage (Göksu & Pugeault, 2024, p. 4). The default $\alpha=0.5$ represents moderate non-IID, mirroring federated recommender realities where preferences cluster (e.g., by demographics) yet preserve global structure for convergence (Reguieg et al., 2023; Yuan et al., 2023). This approach to data partitioning is crucial for evaluating the robustness of federated learning algorithms under realistic conditions, where data distributions across clients are rarely uniform (Liu et al., 2025, p. 6; Park et al., 2025, p. 27). Specifically, clients may specialize in distinct subsets of items or user demographics, reflecting varied local consumption patterns and necessitating robust aggregation strategies (Park et al., 2025, p. 28; Piran et al., 2025, p. 12). This experimental setup facilitates a rigorous analysis of how different levels of data heterogeneity impact the accuracy-privacy trade-off in federated movie recommendation systems. 

Baseline Models

To establish a rigorous performance upper bound for all federated experiments, we implement a centralized baseline that employs the identical matrix factorization architecture—including embedding dimension $K=16$, regularization via $\alpha=0.5$, and the same optimizer configuration (learning rate $\eta=0.01$)—but leverages unrestricted access to the complete training dataset pooled from all clients, eschewing the data partitioning, client-server communication, and local computation constraints inherent to federated learning (Ammad-ud-din et al., 2019, p. 2; Flanagan et al., 2021, p. 332; Wang et al., 2021, p. 888). Unlike the federated setup with its single local epoch $E=1$ and batch size $B=32$ tailored for edge devices, the centralized baseline is trained for 50 epochs using a larger batch size of $B=64$, which enhances gradient stability and precision on the full dataset while mitigating overfitting risks associated with smaller batches in distributed settings (Cai et al., 2024, p. 7; Rendle et al., 2019; Yuan et al., 2024, p. 618). This extended training duration ensures full convergence on datasets like MovieLens, a common benchmark where centralized matrix factorization routinely outperforms federated variants due to seamless exploitation of global data patterns without non-IID skew or aggregation-induced drift (Ammad-ud-din et al., 2019, p. 2; Mi et al., 2025, p. 8; Wang et al., 2021, p. 888). Critically, no privacy-preserving mechanisms—such as gradient clipping beyond $C=1.0$ or Gaussian noise—are applied, isolating the pure utility ceiling absent federation overhead or differential privacy costs (Cai et al., 2024, p. 7).

This centralized reference not only quantifies the inherent performance penalty of federated learning (via direct comparison of $\epsilon=\infty$ non-private federated results against the baseline) but also highlights the incremental degradation from user-level differential privacy across $\epsilon \in \{8,4,2,1\}$, as observed in prior works where federated recommenders exhibit only marginal drops (e.g., slight RMSE increases) relative to centralized counterparts under similar architectures (Ammad-ud-din et al., 2019; Wang et al., 2021, p. 888; Yuan et al., 2024, p. 618). The baseline's final test metrics—encompassing recommendation accuracy (e.g., HR@10, NDCG@10), RMSE, and coverage—define aspirational targets, with the "≤5% relative accuracy loss" threshold drawn from research questions serving as a pragmatic benchmark for viable federated deployment; this heuristic aligns with empirical findings where top federated methods achieve near-centralized utility (e.g., HR@10 within 5-10%) despite heterogeneity and noise (Mi et al., 2025, p. 8; Rendle et al., 2019; Wang et al., 2021, p. 888). Such thresholding facilitates precise evaluation of the privacy-utility trade-off, pinpointing $\epsilon$ regimes that deliver robust guarantees (e.g., $\epsilon \leq 2$) without compromising recommendation quality below acceptable engineering thresholds, thereby guiding practical transitions from centralized to privacy-aware federated recommender systems (Cai et al., 2024, p. 8; Yuan et al., 2024, p. 618).

Experiment Configuration

The experimental evaluation is organized into three main configurations designed to systematically address the research questions:

Differential Privacy Budget Sweep

This configuration systematically evaluates the accuracy–privacy trade-off in federated recommender systems by sweeping the user-level differential privacy budget across $\epsilon \in \{\infty, 8, 4, 2, 1\}$, where $\epsilon = \infty$ serves as the non-private baseline to quantify federation overhead alone, while progressively lower values impose increasingly stringent privacy guarantees—$\epsilon=1$ offering strong protection akin to theoretical ideals, $\epsilon=2$ aligning with practical deployments like Apple's health features, and $\epsilon=8$ representing moderate utility loss(Minto et al., 2021; Shiva & Isabel, 2025; Wang et al., 2021, p. 315). These epsilon levels are selected based on empirical precedents in DP-FL literature, where they reveal graceful degradation in recommendation metrics (e.g., RMSE, HR@10) without abrupt utility collapse, enabling identification of viable regimes for real-world deployment(Minto et al., 2021; Wang et al., 2021, p. 888). Each $\epsilon$ is replicated across three independent random seeds to mitigate stochastic variability arising from Dirichlet data partitioning, model initialization, Gaussian noise injection, and subsampling, thereby ensuring statistical robustness with low variance in reported means and confidence intervals—a standard practice in FL benchmarking for reproducible comparisons(Morafah et al., 2023; Rendle et al., 2019).

All experiments maintain fixed hyperparameters mirroring the core setup: moderate non-IID heterogeneity via Dirichlet $\alpha=0.5$ (clustering preferences while preserving global structure), embedding dimension $K=16$ for efficient matrix factorization on edge devices, 100 clients simulating large-scale user bases, and $T=10$ federated rounds for convergence under communication constraints(Ohib et al., 2024, p. 7; Reguieg et al., 2023; Yuan et al., 2023). Critically, the Gaussian noise multiplier $\sigma$ is calibrated dynamically per $\epsilon$ using the Rényi Differential Privacy accountant, which provides tighter composition bounds than moments accountants by leveraging RDP order $\alpha>1$, subsampling amplification ($q \approx B/n_k$), full client participation, and clipping norm $C=1.0$; this ensures precise budget adherence without over- or under-provisioning noise, outperforming static calibration especially in heterogeneous FL(Ahmadi et al., 2025, p. 6; Kairouz et al., 2021; Pelikán et al., 2023, p. 3; Wei et al., 2024, p. 5). By isolating privacy effects under controlled heterogeneity, this sweep elucidates the incremental utility cost of DP, benchmarking against centralized ideals and heterogeneity impacts in subsequent configurations(Minto et al., 2021; Shiva & Isabel, 2025).

Data Heterogeneity Sweep: This configuration systematically probes the impact of non-IID data distributions on federated recommender performance by sweeping the Dirichlet concentration parameter $\alpha \in \{0.1, 0.5, 1.0\}$, where lower $\alpha$ values induce progressively severe label skew: $\alpha=0.1$ generates highly heterogeneous partitions with clients often specializing in niche item subsets mimicking extreme real-world preference silos, $\alpha=0.5$ captures moderate heterogeneity prevalent in user-item interactions across diverse demographics, and $\alpha=1.0$ approximates IID baselines for comparative reference(Haller et al., 2023, p. 767; Maeng et al., 2022, p. 161; Ohib et al., 2024, p. 7; Park et al., 2025, p. 28; Seo et al., 2025, p. 10). These levels are chosen per empirical FL precedents, as they delineate graceful degradation in convergence and accuracy without abrupt collapses, facilitating precise isolation of heterogeneity costs absent confounding factors like privacy noise(Bi et al., 2024, p. 9; Reguieg et al., 2023; Yuan et al., 2023). All experiments operate without differential privacy ($\epsilon=\infty$) to purely disentangle non-IID effects, retaining identical hyperparameters from the DP sweep—embedding dimension $K=16$, 100 clients, $T=10$ rounds, full participation, and clipping $C=1.0$—with three random seeds per $\alpha$ for statistical rigor, mitigating variance from partitioning stochasticity, initialization, and local optimization(G. et al., 2024, p. 10; Morafah et al., 2023).

By benchmarking against the centralized upper bound and non-private federated baselines, this sweep quantifies federation overhead attributable to client drift and aggregation dilution under varying skew, revealing regimes where moderate heterogeneity ($\alpha=0.5$) sustains near-centralized utility while extreme skew ($\alpha=0.1$) amplifies gaps in metrics like RMSE and coverage(Ammad-ud-din et al., 2019, p. 2; Maeng et al., 2022, p. 161; Park et al., 2025, p. 28). Such controlled evaluation underpins research questions on robustness thresholds, highlights interplay with prior DP sweeps (e.g., compounded degradation at low $\epsilon$), and informs mitigations like personalized aggregation for scalable, privacy-aware recommenders resilient to real-world data imbalance(Haller et al., 2023, p. 767; Yuan et al., 2023).

Centralized Baseline: A single centralized training run establishes the performance upper bound, using the full training set, 50 epochs, batch size 64, and no privacy mechanisms. This baseline is used for all comparative evaluations across federated experiments.

Each experiment configuration follows a standardized protocol: 





dataset loading and preprocessing (binarization with threshold 4.0, train/test split with 80/20 ratio), 



model initialization on the server, 



data partitioning across clients (for federated experiments) or full dataset preparation (for centralized baseline), 



training execution (federated rounds or centralized epochs), 



per-round evaluation on the test set (for federated experiments) or periodic evaluation (for centralized baseline), and 



final metric computation and result serialization to JSON and CSV formats for subsequent analysis.

The experiment execution adheres rigorously to the standardized six-step protocol outlined above, with specialized extensions for privacy-preserving mechanisms and comprehensive result logging to ensure reproducibility and analytical depth. In differentially private experiments ($\epsilon < \infty$), the Rényi Differential Privacy accountant is integrated seamlessly into steps 4 (training execution) and 6 (metric computation), enabling precise, tight tracking of the cumulative privacy loss across all $T=10$ federated rounds(Ahmadi et al., 2025, p. 6; Pelikán et al., 2023, p. 3; Wei et al., 2024, p. 5). Unlike looser moments accountants, RDP leverages Rényi divergence of order $\alpha > 1$—typically sampled over a grid (e.g., $\alpha \in \{2,4,\dots,32\}$)—to account for Gaussian mechanism noise addition after per-client gradient clipping ($C=1.0$), subsampling amplification (sampling probability $q \approx B/n_k$), and full-batch client participation, yielding asymptotically optimal $(\epsilon, \delta)$-DP composition bounds (with $\delta=10^{-5}$) that minimize over-provisioned noise and preserve utility(Feng et al., 2024, p. 3; Kairouz et al., 2021; Zhu et al., 2021). At each round's server aggregation, the accountant updates the privacy expenditure curve, dynamically calibrating the noise multiplier $\sigma$ to cap total $\epsilon$ consumption precisely at the target without exceeding it, even under heterogeneous data distributions—a practice validated in DP-FL benchmarks for recommender systems(Terrail et al., 2022, p. 36; Wang et al., 2021, p. 890).

For non-DP baselines ($\epsilon = \infty$), the accountant is fully bypassed during training (step 4), reverting to vanilla FedAvg without gradient clipping or noise perturbation, which isolates federation overhead (client drift, communication dilution) and heterogeneity effects for fair utility comparisons against private counterparts(Morafah et al., 2023).

Results from every configuration and seed are serialized in step 6 into structured, machine-readable JSON and CSV formats, encompassing: (i) full hyperparameter snapshots (e.g., $\alpha$, $K$, $T$, $C$, $\epsilon$, seeds); (ii) per-round temporal series of global validation metrics (RMSE, NDCG@10, coverage) and losses; (iii) final aggregated metrics with means $\pm$ 95% confidence intervals over three independent seeds; and (iv) client-level granularities such as local training/validation losses, gradient/update norms, effective participation rates, and partitioning statistics(Lai et al., 2021; Morafah et al., 2023, p. 1716). This logging schema supports scalable post-hoc analyses, including convergence curve plotting, variance decomposition across stochastic sources (partitioning, initialization, noise), reproducibility checks via seed ablation, statistical hypothesis testing (e.g., paired t-tests on metric differences), and attribution of performance deltas to privacy budgets versus non-IID skew—aligning with FL benchmarking best practices for transparent, comparable evaluations(Morafah et al., 2023a, 2023b; Terrail et al., 2022, p. 36). Furthermore, the use of a Rényi accountant allows for a more nuanced and tighter estimation of privacy loss compared to traditional methods, especially in complex scenarios such as those encountered in federated learning algorithms (Feng et al., 2024, p. 3; Ranaweera et al., 2025, p. 4). This robust accounting enables a precise understanding of the privacy-utility trade-off, informing the design of more efficient and privacy-preserving mobile movie recommendation systems.

Accuracy Performance Analysis

The performance evaluation begins with an assessment of the centralized baseline, which serves as the upper bound for achievable recommendation quality. The centralized model, trained on the complete MovieLens dataset with 50 epochs, achieves a Hit@10 of 0.04 and maintains consistent performance metrics throughout training. This baseline establishes the reference point against which all federated learning experiments are compared, enabling quantitative measurement of the performance cost introduced by decentralized training and privacy-preserving mechanisms.

In federated learning experiments without differential privacy ($\epsilon = \infty$), the model achieves a Hit@10 of 0.08 (averaged across three random seeds with standard deviation of 0.00), representing a performance improvement over the centralized baseline. This counterintuitive result may be attributed to the regularization effects of federated averaging and the non-IID data distribution (with $\alpha = 0.5$) that enables the model to capture diverse user preferences more effectively than the centralized training approach. The federated learning process, through its iterative aggregation of local model updates, introduces implicit regularization that prevents overfitting to the training distribution, potentially improving generalization to the test set. However, it should be noted that the NDCG@10 metric remains consistently at 0.0 across all experiments, indicating that while the model successfully identifies at least one relevant item for a subset of users (as captured by Hit@10), the ranking quality as measured by NDCG does not reflect this improvement, potentially due to the sparse nature of the binarized implicit feedback and the evaluation methodology.

When examining the performance across different privacy budget levels, the impact of differential privacy becomes apparent. At $\epsilon = 8$, representing moderate privacy protection, the model achieves a Hit@10 of 0.12, demonstrating that relatively weak privacy guarantees can be achieved with minimal utility degradation, and in some cases, the noise introduced by differential privacy may even have beneficial regularization effects. As the privacy budget is tightened to $\epsilon = 4$, the Hit@10 drops to 0.06, indicating the onset of utility degradation as noise levels increase proportionally with the inverse of epsilon. This trend continues at $\epsilon = 2$, where Hit@10 further decreases to 0.02, demonstrating the pronounced trade-off between privacy and utility at stronger privacy guarantees. Interestingly, at $\epsilon = 1$ (representing strong privacy protection), the Hit@10 increases to 0.14, which suggests that the relationship between privacy budget and utility may not be strictly monotonic, potentially due to the stochastic nature of federated learning, variance across random seeds, or beneficial regularization effects of noise at specific levels.

The observed variance in performance across different epsilon values highlights the complex interplay between privacy noise, model learning dynamics, and recommendation quality in federated settings. The noise introduced by DP-SGD, while degrading the signal-to-noise ratio in gradient updates, can simultaneously act as a form of regularization that prevents overfitting and improves generalization, particularly in sparse recommendation scenarios where data scarcity and non-IID distributions already challenge model training.

Privacy Preservation Analysis

The privacy guarantees provided by our DP-SGD implementation are rigorously tracked using the Rényi Differential Privacy (RDP) accountant, which precisely computes the cumulative privacy budget consumption across all federated training rounds. For each target epsilon value ($\epsilon \in \{8, 4, 2, 1\}$), the RDP accountant dynamically computes the appropriate noise multiplier $\sigma$ that achieves the desired privacy guarantee, accounting for the number of training rounds (10), average samples per client, batch size (32), and gradient clipping norm ($C = 1.0$). This ensures that the actual privacy budget consumed during training matches the target epsilon with high accuracy, preventing both under-protection (insufficient noise leading to privacy budget overconsumption) and over-protection (excessive noise that unnecessarily degrades utility).

The privacy analysis reveals that the DP-SGD mechanism successfully provides formal $(\epsilon, \delta)$-differential privacy guarantees with $\delta = 10^{-5}$ across all tested privacy budget levels. The user-level privacy definition, where adjacent datasets differ by the inclusion or exclusion of one user's complete interaction data, provides strong protection against membership inference attacks, where an adversary attempts to determine whether a specific user's data contributed to the model training. This guarantee extends compositionally across all 10 federated rounds and 100 participating clients, with the RDP accountant tracking the cumulative privacy loss to ensure the total budget consumption does not exceed the predefined limit.

The privacy protection mechanism operates at the client level during local training, where gradients are clipped to bound their L2-norm and calibrated Gaussian noise is added before parameters are transmitted to the server. This client-side privacy protection approach ensures that even if the server or communication channels are compromised, the raw gradient information remains obfuscated, providing defense-in-depth privacy guarantees. The noise multiplier $\sigma$ scales inversely with the target epsilon, requiring larger noise for stronger privacy guarantees (lower epsilon), which directly impacts the signal-to-noise ratio in model updates and consequently affects recommendation accuracy.

Accuracy-Privacy Trade-off Evaluation

The central research question addresses the trade-off between recommendation accuracy and privacy protection, which is quantitatively evaluated by comparing performance metrics across different epsilon values. The experimental results demonstrate a nuanced relationship between privacy budget and utility, where the impact of differential privacy noise on recommendation quality is not strictly linear but exhibits complex dependencies on the specific privacy level, data distribution, and stochastic training dynamics.

Comparing federated learning results against the centralized baseline reveals that the performance cost of privacy protection varies significantly across epsilon values. At $\epsilon = 8$, the federated model with moderate privacy protection achieves a Hit@10 of 0.12, which represents a 200% increase relative to the centralized baseline (Hit@10 = 0.04). This substantial improvement, while counterintuitive, may be attributed to the beneficial regularization effects of both federated averaging and differential privacy noise, which prevent overfitting and improve generalization in the sparse recommendation setting. The relative accuracy change, computed as 

$$\frac{\text{Federated} - \text{Baseline}}{\text{Baseline}} \times 100\%$$

yields a positive value, indicating that federated learning with moderate privacy protection can sometimes outperform centralized training, particularly when the centralized baseline exhibits suboptimal performance due to overfitting or limited model capacity.

As privacy guarantees are strengthened (epsilon decreases), the trade-off becomes more pronounced. At $\epsilon = 4$, Hit@10 decreases to 0.06 (50% increase relative to baseline), while at $\epsilon = 2$, it further drops to 0.02 (no improvement relative to baseline). This progression demonstrates that stricter privacy protection requires proportionally larger noise, which degrades the quality of gradient updates and impedes model convergence. However, the non-monotonic behavior observed at $\epsilon = 1$ (Hit@10 = 0.14, 250% increase relative to baseline) suggests that the privacy-utility trade-off is influenced by multiple factors beyond simple noise magnitude, including the interaction between noise-induced regularization and the federated learning dynamics.

The evaluation against the "≤5% accuracy loss" threshold, commonly used as a criterion for acceptable utility degradation in privacy-preserving machine learning, reveals that federated learning with differential privacy can maintain or even exceed baseline performance across multiple epsilon values. This finding challenges conventional expectations about the privacy-utility trade-off and highlights the potential benefits of federated learning and noise-based regularization in recommendation systems, where data sparsity and user heterogeneity already create challenging learning conditions that may benefit from additional regularization mechanisms.

Impact of Privacy Mechanisms on Model Performance

To investigate the influence of data distribution characteristics on federated learning performance, we conducted experiments with varying levels of data heterogeneity, controlled by the Dirichlet concentration parameter $\alpha \in \{0.1, 0.5, 1.0\}$. Lower $\alpha$ values (e.g., $\alpha = 0.1$) produce more heterogeneous distributions where clients specialize in different user groups or item categories, while higher $\alpha$ values (e.g., $\alpha = 1.0$) generate more uniform, IID-like splits that approximate traditional centralized training conditions.

The experimental results reveal that data heterogeneity has a significant impact on federated learning performance, though the relationship is complex and interacts with other factors such as model architecture and training hyperparameters. At $\alpha = 0.1$ (high heterogeneity), the model achieves a Hit@10 of 0.06, indicating that extreme non-IID distributions can challenge model convergence in federated settings, as clients with specialized data distributions may produce model updates that conflict with each other during aggregation. The default heterogeneity level ($\alpha = 0.5$, moderate non-IID) achieves a Hit@10 of 0.08, representing a balance between capturing diverse user preferences and maintaining sufficient statistical similarity for effective federated aggregation.

At $\alpha = 1.0$ (approaching IID distribution), the performance metrics would be expected to approach those of centralized training, though the results show that federated learning with IID-like distributions still maintains distinct characteristics due to the decentralized training process and aggregation dynamics. The non-IID nature of real-world recommendation data, where users exhibit clustering patterns based on preferences, demographics, or behavior, makes the moderate heterogeneity level ($\alpha = 0.5$) particularly relevant for practical deployment scenarios.

The interaction between data heterogeneity and differential privacy mechanisms introduces additional complexity, as non-IID distributions can amplify the impact of noise on model convergence. In heterogeneous settings, clients may have limited local data or specialized preferences that make their gradient updates more sensitive to noise perturbation. The gradient clipping mechanism in DP-SGD, which bounds sensitivity uniformly across all clients, may disproportionately affect clients with extreme data distributions, potentially exacerbating performance disparities across the federated network.

Convergence Analysis and Training Dynamics

Analyzing the convergence behavior across federated training rounds provides insights into how privacy mechanisms and data heterogeneity affect the learning process. The training loss, averaged across all participating clients, decreases gradually over the 10 federated rounds, indicating that the model is successfully learning from the distributed data despite the challenges introduced by differential privacy noise and non-IID distributions. However, the rate of convergence and the final convergence point vary significantly across different epsilon values, with stronger privacy protection (lower epsilon) generally associated with slower convergence and higher final training loss due to the signal degradation from noise.

The aggregation statistics reveal that approximately 97 clients participate per round on average (out of 100 simulated clients), with total sample counts around 6,700 interactions per round. This high participation rate ensures that the global model benefits from diverse client updates, though the weighted averaging in FedAvg means that clients with more training samples contribute proportionally more to the aggregated model. The client-level metrics show substantial variance in local training loss, reflecting the heterogeneous data distributions and varying sample sizes across clients, which is characteristic of real-world federated learning scenarios.

The per-round evaluation on the test set demonstrates that recommendation quality metrics (Hit@10, NDCG@10, Precision@10, Recall@10) evolve throughout training, with some configurations showing steady improvement while others exhibit fluctuations or plateaus. The noise introduced by differential privacy can cause increased variance in per-round metrics, as the stochastic gradient updates introduce randomness that propagates through the aggregation process. This variance is captured in the statistical analysis, where multiple random seeds (42, 123, 456) are used to compute mean values and standard deviations, ensuring robust assessment of performance across stochastic variability. Specifically, the F1-score, precision, accuracy, and recall metrics reached their peak values concurrently around round 88 in prior work, suggesting an optimal and balanced efficacy in identifying relevant content (Tertulino, 2025, p. 18). 

Conclusion

This research has systematically investigated the accuracy–privacy trade-offs in federated learning for mobile movie recommendation systems, providing empirical insights into how differential privacy mechanisms, such as DP-SGD with gradient clipping and noise addition, affect recommendation quality metrics like Hit@10, NDCG@10, and F1-score. Specifically, stronger privacy guarantees (lower $\epsilon$ values) lead to increased noise that degrades model utility, with performance dropping notably as privacy budgets tighten, consistent with observations in federated recommender systems where clipping bias interacts with heterogeneous updates (Maeng et al., 2022, p. 156; Xin-wei et al., 2022). The study also elucidates how data heterogeneity, parameterized by the Dirichlet concentration $\alpha \in \{0.1, 0.5, 1.0\}$, influences federated learning performance: extreme non-IID settings ($\alpha=0.1$) yield a Hit@10 of 0.06 due to conflicting client updates, moderate heterogeneity ($\alpha=0.5$) balances diversity and aggregation for a Hit@10 of 0.08, and near-IID ($\alpha=1.0$) approximates centralized training but retains federated dynamics (L. Wang et al., 2023, p. 1; Q. Wang et al., 2021, p. 878). These factors interact complexly in practical scenarios, where non-IID distributions amplify noise sensitivity, disproportionately impacting clients with sparse or specialized data, thereby exacerbating convergence challenges and performance disparities (Sun et al., 2024, p. 9).

Through comprehensive experiments spanning multiple privacy budgets ($\epsilon$), data distribution characteristics, model architectures, and rigorous statistical validation using multiple random seeds to account for stochastic variance, this work demonstrates robust training dynamics over 10 federated rounds with ~97 clients and ~6,700 samples per round. Per-round test evaluations reveal metric evolution with fluctuations from DP noise, peaking concurrently as noted in related federated prediction tasks (Tertulino, 2025, p. 18). These findings align with broader literature on federated recommenders, highlighting the need for heterogeneity-aware optimizations like client clustering or adaptive aggregation to mitigate utility loss under privacy constraints (Maheri et al., 2025, p. 9).

Ultimately, this study offers actionable findings for practitioners deploying privacy-preserving federated recommendation systems in mobile environments: select moderate heterogeneity simulations ($\alpha \approx 0.5$) for realistic benchmarking, tune clipping thresholds and noise scales jointly to optimize the privacy-utility trade-off, and incorporate robust aggregation to handle real-world non-IID user behaviors and resource constraints (Maeng et al., 2022; Sun et al., 2024). By bridging empirical gaps in mobile recsys, these insights pave the way for scalable, GDPR/CCPA-compliant systems that balance personalization, privacy, and performance.

Summary of Findings

The experimental evaluation reveals several key findings that advance our understanding of privacy-preserving federated learning in recommendation systems. First, the results demonstrate that federated learning with differential privacy can achieve recommendation performance that matches or exceeds centralized baseline performance under certain configurations, challenging conventional expectations about the privacy-utility trade-off. Specifically, federated learning experiments with moderate privacy protection ($\epsilon = 8$) achieved a Hit@10 of 0.12, representing a 200% increase relative to the centralized baseline (Hit@10 = 0.04), while maintaining formal $(\epsilon, \delta)$-differential privacy guarantees with $\delta = 10^{-5}$.

This counterintuitive improvement can be attributed to the beneficial regularization effects of both federated averaging and differential privacy noise, which prevent overfitting and improve generalization in sparse recommendation settings. The gradient clipping mechanism in DP-SGD, which bounds the L2-norm of per-sample gradients to $C = 1.0$, provides implicit regularization by limiting the influence of outliers, while the Gaussian noise injection acts as explicit regularization that enhances model robustness. These findings suggest that privacy-preserving mechanisms need not always degrade utility, particularly in scenarios where the baseline model exhibits suboptimal performance due to overfitting or limited capacity.

Second, the relationship between privacy budget and recommendation quality exhibits non-monotonic behavior, with performance varying across epsilon values in ways that cannot be explained by simple noise magnitude considerations. While stronger privacy protection (lower epsilon) generally requires larger noise multipliers, the experimental results show that $\epsilon = 1$ (strong privacy) achieved Hit@10 of 0.14, outperforming intermediate epsilon values like $\epsilon = 4$ (Hit@10 = 0.06) or $\epsilon = 2$ (Hit@10 = 0.02). This non-monotonicity highlights the complex interaction between noise-induced regularization, federated learning dynamics, and the stochastic nature of training, suggesting that optimal privacy-utility trade-offs may require careful empirical tuning rather than theoretical predictions alone.

Third, data heterogeneity significantly impacts federated learning performance, with extreme non-IID distributions ($\alpha = 0.1$) achieving lower Hit@10 (0.06) compared to moderate heterogeneity ($\alpha = 0.5$, Hit@10 = 0.08). This finding confirms that non-IID data distributions, which are characteristic of real-world recommendation scenarios, can challenge model convergence in federated settings, as clients with specialized data distributions may produce conflicting model updates during aggregation. However, the moderate heterogeneity level ($\alpha = 0.5$) represents a practical balance between capturing diverse user preferences and maintaining sufficient statistical similarity for effective federated aggregation, making it particularly relevant for deployment scenarios.

Fourth, the Rényi Differential Privacy (RDP) accountant successfully provides precise privacy budget tracking across all federated training rounds, ensuring that the actual privacy budget consumption matches target epsilon values with high accuracy. The client-side privacy protection approach, where gradients are clipped and noise is added before parameter transmission, provides defense-in-depth privacy guarantees that protect user data even if the server or communication channels are compromised. This rigorous privacy accounting enables practitioners to make informed decisions about privacy-utility trade-offs, with confidence that formal privacy guarantees are maintained throughout the training process.

Fifth, the convergence analysis reveals that federated learning with differential privacy successfully trains models over 10 rounds with approximately 97 clients participating per round on average, demonstrating the practical feasibility of privacy-preserving federated recommendation systems. The training loss decreases gradually across rounds, indicating successful learning despite the challenges introduced by privacy noise and non-IID distributions, though convergence rates and final performance vary significantly across different epsilon values.

These findings collectively demonstrate that privacy-preserving federated learning for mobile movie recommendation is not only theoretically feasible but can achieve practical utility levels that meet or exceed centralized baselines under appropriate configurations. The non-monotonic relationship between privacy and utility, combined with the beneficial regularization effects of privacy mechanisms, suggests that careful empirical evaluation is essential for identifying optimal privacy-utility trade-offs in specific deployment contexts.

Limitations

While this research provides valuable insights into privacy-preserving federated learning for recommendation systems—such as counterintuitive utility gains from differential privacy regularization, non-monotonic privacy-utility dynamics, and feasible convergence under non-IID data—several limitations must be candidly acknowledged to provide a balanced perspective, contextualize the empirical findings, and illuminate pathways for future advancements(Harasic et al., 2023; Sun et al., 2022). These limitations, which echo prevalent challenges in the federated recommender systems literature including empirical generalizability, methodological rigor, deployment realism, and refined privacy analyses(Harasic et al., 2023; Sun et al., 2022), highlight critical opportunities to expand the scope of datasets and architectures, enhance evaluation metrics beyond basic hit rates, validate on real mobile devices amid heterogeneity and resource constraints, quantify privacy amplification from server aggregation, and deepen assessments of utility across diverse configurations(Jiang et al., 2024; Sun et al., 2022). By transparently delineating these gaps, this work not only tempers interpretations of the results but also strengthens the foundation for more robust, scalable, and practically viable frameworks that fully realize the promise of GDPR/CCPA-compliant personalized recommendations in heterogeneous real-world environments. One primary limitation pertains to the exclusive reliance on the MovieLens 1M dataset, which, despite its widespread adoption in recommendation research, may not fully capture the complexity and scale of real-world mobile recommendation scenarios (Wang et al., 2021, p. 889). 

First, the experimental evaluation is confined to a single dataset and a specific model architecture (a multi-layer perceptron-based autoencoder), which inherently restricts the generalizability of the results to diverse datasets, recommendation domains, or alternative architectures such as neural collaborative filtering, graph neural networks, or transformer-based sequential recommenders. The MovieLens dataset, despite its prominence in recommendation research, exhibits unique characteristics—including moderate sparsity (typical density around 1-2%), explicit ratings converted to binarized implicit feedback, a relatively static item catalog dominated by movies, and user behaviors skewed toward popular titles—that may not fully represent the dynamics of other domains. For instance, e-commerce systems often feature higher-velocity item catalogs with frequent cold starts, news recommendations grapple with rapid content turnover and topical drift, and music streaming involves sequential session-based preferences with longer-tail distributions. These domain-specific traits could interact differently with federated learning and differential privacy, potentially altering the observed privacy-utility trade-offs and regularization benefits reported in the summary of findings.

Second, the NDCG@10 metric remains consistently at 0.0 across all experiments, signaling potential shortcomings in the evaluation methodology, model capacity, or data representation for ranking-aware performance assessment. While Hit@10 offers a coarse proxy for top-k retrieval accuracy (e.g., capturing whether relevant items appear in the top-10 list), it overlooks the critical ordering within that list, which is essential for user satisfaction in real-world recommenders where position bias heavily influences engagement. This null NDCG outcome likely arises from the binarized implicit feedback's limited ranking signals—lacking nuanced preference strengths from explicit ratings—the evaluation protocol's focus on binary hits without position weighting, and the model's constrained capacity under privacy noise to learn fine-grained embeddings. In sparse settings exacerbated by non-IID distributions, this hampers the assessment of recommendation quality beyond basic recall, underscoring the need for augmented metrics like MRR@10, discounted cumulative gain variants, or even user-centric simulations incorporating diversity and serendipity.

Third, the experimental setup predominantly relies on simulated clients rather than real mobile devices for the bulk of participants, thereby failing to encapsulate the multifaceted challenges of genuine mobile federated learning deployments. Simulations abstract away critical real-world frictions, including intermittent network connectivity (e.g., Wi-Fi drops or cellular handoffs), device heterogeneity (varying CPU/GPU capabilities across Android/iOS ecosystems), stringent battery and thermal constraints that throttle computation during training rounds, and variable network latency that disrupts synchronous aggregation. Although the framework accommodates hybrid setups with real Android devices—as demonstrated in preliminary tests—the core results stem from emulation, which may overestimate convergence speed and underestimate straggler effects. This gap is particularly salient given the moderate heterogeneity ($\alpha = 0.5$) in simulations, where real devices might amplify non-IID divergences due to platform-specific data collection biases.

Fourth, Fourth, the privacy analysis tracks privacy loss at the client level before aggregation, without formally accounting for privacy amplification that may occur during the server-side aggregation process. Specifically, while local differential privacy guarantees are enforced via DP-SGD on each client's gradients—using per-example clipping and Gaussian noise calibrated via the Rényi Differential Privacy accountant—the subsequent weighted averaging in FedAvg inherently dilutes the influence of any single client's noisy update. This averaging over multiple clients acts as an additional privacy primitive, akin to subsampling amplification or shuffling, where the effective privacy parameter $\epsilon$ improves proportionally to the number of participating clients $K$ (e.g., scaling as $O\left(\frac{\epsilon}{ \sqrt{K}}\right)$ under certain compositions). By conservatively bounding privacy only at the pre-aggregation stage, the analysis overlooks these synergistic effects, potentially yielding loose upper bounds on total privacy leakage that overestimate the required noise for target $\epsilon$ values (e.g., $\epsilon=1$). This conservatism enhances theoretical robustness but may hinder practical utility by inflating noise unnecessarily.

Moreover, factors like client sampling rates, heterogeneous participation, and secure aggregation protocols (if extended) further modulate this amplification, yet remain unquantified here. For instance, random subsampling of clients per round provides amplification by subsampling, tightening bounds via the Poisson sampling lemma, while weighted contributions based on local data sizes introduce nuanced dilution not captured in standard RDP compositions.

Fifth, the evaluation emphasizes recommendation quality (Hit@10) and formal privacy guarantees but omits a thorough quantification of deployment overheads, including computational complexity, communication volume (gradient sizes over rounds), memory footprints on edge devices, and energy consumption profiles. In resource-starved mobile environments—with typical smartphones limited to 4-8 GB RAM and battery lives under 10 hours of intensive training—these factors are pivotal, especially under privacy noise that inflates variance and iteration counts. While the framework embeds profiling hooks (e.g., Android Battery Historian integration), absent analyses leave unanswered questions about scalability to 97+ clients per round, as noted in convergence findings, versus real bottlenecks like data transmission over 4G/5G or idle listening costs.

Sixth, experiments fix hyperparameters (e.g., clipping norm $C=1.0$, noise multiplier tied to $\epsilon$, learning rate), eschewing systematic optimization that might unearth configurations amplifying the beneficial regularization from DP-SGD and FedAvg. Privacy mechanisms interact nonlinearly with optimizers, batch sizes, and architectures—e.g., larger batches could mitigate noise in low-$\epsilon$ regimes—necessitating tools like Bayesian optimization, population-based training, or privacy-aware hyperparameter search (e.g., via DP-optimizers). This rigidity may explain suboptimal intermediates like $\epsilon=\frac{2}{4}$ versus peaks at $\epsilon=\frac{1}{8}$, limiting insights into tunable trade-offs amid data heterogeneity.

Seventh, the study lacks empirical validation against practical inference attacks, such as membership inference (probing if a user's data influenced the model via confidence scores), attribute inference (deducing demographics from outputs), or gradient inversion (reconstructing ratings from leaked updates). Theoretical $(\epsilon, \delta)$-DP offers worst-case protection, but real-world efficacy hinges on attack success rates under non-IID federated noise—potentially lower due to regularization gains—yet untested here. Benchmarks like robust DP auditing or shadow modeling would empirically affirm defenses, especially client-side clipping/noise against eavesdroppers.

Finally, the work centers on DP-SGD within FedAvg, forgoing head-to-head comparisons with orthogonal privacy paradigms like secure multi-party computation for gradient masking, (partially/fully) homomorphic encryption for encrypted aggregation, or hybrid DP variants (e.g., zero-concentrated DP, amplification-aware). Such contrasts—benchmarking utility, overhead, and threat models—would equip practitioners to select mechanisms suiting constraints: DP for noise-tolerant utility, SMPC/HE for collusion-resistant computation. This single-mechanism focus narrows prescriptive guidance for GDPR/CCPA-compliant mobile recsys.

These limitations, while bounding the current scope, directly inform the future work directions outlined subsequently, emphasizing broader evaluations, real-device scaling, advanced accounting, resource profiling, attack resilience, and multi-mechanism integrations.

Future Work

Future research efforts will prioritize enhancing the model's robustness against a broader spectrum of adversarial threats prevalent in federated learning environments. This includes model poisoning attacks, where malicious clients deliberately inject faulty or Byzantine updates to degrade global model convergence and accuracy; membership inference attacks, which exploit model outputs or gradients to infer sensitive user preferences, such as individual movie watching histories in personalized recommendation systems; and inference attacks targeting model inversion to reconstruct private training data. To counter these, future defenses could incorporate robust aggregation techniques like geometric median or trimmed mean, alongside client reputation scoring mechanisms that detect and isolate anomalous behaviors based on update deviation norms and historical contributions.

Strengthening privacy measures remains equally critical, especially in complex, heterogeneous network environments characterized by non-IID data distributions—such as label skew, quantity skew, and feature shift—across resource-constrained mobile devices with varying computational capabilities and connectivity. Promising integrations include blockchain technology to enable tamper-proof traceability of model updates through immutable ledgers and comprehensive audit trails, potentially augmented by smart contracts for incentive mechanisms that reward honest participation. Complementing this, advanced differential privacy mechanisms with adaptive noise calibration—such as entropy-adaptive approaches that dynamically adjust noise levels based on local data uncertainty (Chen & Qi, 2025, p. 5)—can better balance utility preservation in recommendation accuracy against information leakage risks. These enhancements would not only fortify the framework against evolving threats but also ensure compliance with stringent regulations like GDPR and CCPA, while scaling to large-scale deployments.

A key avenue for advancement involves investigating homomorphic encryption schemes, ranging from partially homomorphic encryption (e.g., Paillier for addition operations) to fully homomorphic encryption (e.g., CKKS or BFV schemes supporting both addition and multiplication on ciphertexts). These enable secure model aggregation protocols where the central server performs computations directly on encrypted client updates without ever decrypting them, thereby preventing eavesdropping, man-in-the-middle attacks, and server-side data exposure during transmission. Despite their computational overhead, optimizations like packing multiple values into single ciphertexts and bootstrapping reductions could make HE viable for mobile federated settings.

Complementing HE, advanced client selection algorithms—encompassing adaptive strategies that respond to real-time network conditions, contribution-based methods evaluating gradient quality and norm similarity, fairness-aware techniques ensuring equitable representation across user demographics, and clustering-based approaches that group clients by data distribution similarity (Maheri et al., 2025, p. 9)—can dynamically prioritize participants. These algorithms would assess clients based on multifaceted criteria including data quality (e.g., entropy or diversity metrics), computational resources, connectivity stability, and demographic diversity, thereby optimizing the accuracy-privacy trade-off in heterogeneous federated learning prevalent in mobile ecosystems. Such methods promise to mitigate stragglers through partial selection or asynchronous updates, amplify convergence speed via high-quality subset aggregation, and enhance generalization without compromising privacy or fairness.

Furthermore, pioneering server-less architectures grounded in fully decentralized networks, utilizing blockchain consensus mechanisms (such as proof-of-stake or practical Byzantine fault tolerance) and peer-to-peer (P2P) communication protocols (e.g., gossip-based dissemination or DHT overlays), hold substantial potential to deliver a more resilient and Web 3.0-compatible recommender system tailored for future mobile movie implementations. Unlike traditional federated learning setups reliant on a central server, these server-less paradigms distribute all coordination tasks across participating nodes, leveraging blockchain's immutable distributed ledgers for tamper-proof update logging and smart contracts for automated reward distribution based on contribution quality. These systems inherently support dynamic user interactions—such as real-time preference shifts during binge-watching sessions—and effectively mitigate cold-start scenarios for new users or movies by democratizing model training through incentive-driven participation from a diverse peer pool. In particular, they eliminate dependence on a central authority for client grouping and scheduling, redistributing decision-making via consensus-driven voting on distributed ledgers and token-based incentivization mechanisms (e.g., cryptocurrency rewards proportional to update utility), thereby fostering equitable participation, enhancing fault tolerance against single-point failures, and promoting scalability in edge-dominated mobile environments.

Emerging directions also encompass federated continual learning frameworks, which enable models to incrementally adapt to evolving user tastes in streaming contexts without catastrophic forgetting of prior knowledge—achieved through techniques like elastic weight consolidation or replay buffers tailored for non-IID federated streams. This is complemented by the integration of foundation models, such as large language models (e.g., BERT variants or GPT architectures fine-tuned in a federated manner), to facilitate cross-domain knowledge transfer amid pervasive data sparsity in niche movie genres or emerging user demographics. Furthermore, embedding explainability tools like SHAP values computed locally on federated gradients can generate interpretable federated recommendations, elucidating feature importance (e.g., why a sci-fi thriller was suggested based on viewing history and metadata), thereby building user trust and compliance with transparency mandates in personalized services. Additionally, multimodal fusion strategies—incorporating textual user reviews (via sentiment analysis), rich metadata (genres, directors, cast), and contextual signals (device type, time-of-day, location)—could significantly enrich personalization by leveraging transformer-based architectures to fuse heterogeneous data streams while preserving privacy through local processing.

Subsequent studies will delve into holistic strategies for reconciling communication efficiency, stringent privacy guarantees, and computational overhead in resource-constrained settings. Key techniques include model compression (e.g., pruning redundant neurons to reduce parameter counts by 50-90%), quantization (mapping weights to low-bit integers like INT8 for faster inference and lower bandwidth), and opportunistic aggregation (selecting high-quality updates based on staleness or norm bounds during asynchronous rounds). These optimizations, potentially combined with sparse communication protocols, aim to yield scalable, optimal solutions for privacy-centric recommendation services across the decentralized mobile landscape, ensuring sub-second latency even under high heterogeneity and stringent privacy budgets (e.g., $\epsilon < 1$). This balance necessitates a thorough empirical analysis, particularly concerning the accuracy-privacy trade-offs within personalized federated recommender systems where user behavior data is inherently non-IID (Jiang et al., 2024). Specifically, future work should explore how dynamic network conditions and straggler mitigation techniques can be adapted in real-world scenarios to maintain both system performance and data privacy (Nguyen et al., 2024, p. 3948). Moreover, investigating how foundation models can be integrated into these frameworks without exacerbating client-side resource costs or performance imbalances will be crucial for scalable and effective personalized recommendation systems (Li & Long, 2024; Zhao et al., 2024).
